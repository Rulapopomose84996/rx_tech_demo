- 结果层会显示“当前 PRT 覆盖”，用于判断当前 PRT 是仍在接收中，还是已经出现真实缺包/乱序现象

`scripts/compile/server_shared_cache.sh` 等价于：

执行位置：Linux server `/home/devuser/WorkSpace/rx_tech_demo`


你明天可以直接说：

- “项目采用 CMake + Ninja 构建”
- “服务器侧通过共享 third-party cache 构建”
- “权威构建不在 Windows 上做”

## 9. 运行结果去哪里找

现在每次运行都会生成独立的时间戳目录，格式：

- `20260403_103012_xxx`

例如：

- `results/20260403_103012_dpdk_single_face/`
- `/data/rx_tech_demo/raw_frames/20260403_103012_dpdk_single_face/`

目录职责：

- `results/`：当前接收主线的运行结果目录
- `data/`：样例和参考数据目录，不是当前主线运行输出
- `stdout/`：历史遗留目录，当前主线不再使用
