# C表 回传数据表文档

## 1. 适用范围

C 表用于定义前端设备向上位机回传的业务数据格式，分为以下两类独立包体：

1. **C1 自然状态回传包体**<br>用于周期性回传设备在线状态、工作状态、硬件健康状态、故障与告警信息，以及姿态与倾角传感器观测量。<br>对应 B 通用包头中的 `包类型 = 0x03`。
2. **C2 波位数据回传包体**<br>用于回传波位级采样结果，采用“数据控制区 + IQ 数据长度 + IQ 数据内容”的结构。<br>对应 B 通用包头中的 `包类型 = 0x04`。

本文件统一描述 C1 与 C2 两种包体，但二者**不在同一帧中同时出现**，而是分别由不同 `包类型` 承载。

## 3. C1 自然状态回传包体

### 3.1 整帧结构

```text
C1 整帧结构
├─ B通用包头（固定 24B）
├─ C1 自然状态回传包体（固定 52B）
└─ B通用包尾（固定 8B）
```

### 3.2 长度关系

1. `B通用包头长度 = 24B`
2. `C1包体长度 = 52B`
3. `B通用包尾长度 = 8B`

因此：

**C1整包长度 = 24 + 52 + 8 = 84B**

### 3.3 包体结构说明

C1 自然状态回传包体固定长度为 **52B**，由以下 4 个区段组成：

1. 状态回传控制区：`8B`
2. 设备运行状态区：`16B`
3. 故障告警与计数区：`12B`
4. 姿态与倾角传感器区：`16B`

3. 若后续需要增加更多姿态量，如 Roll、姿态融合角、传感器温度等，应通过版本升级或长度扩展明确新增，不回改当前结构定义。

## 4. C2 波位数据回传包体

### 4.1 整帧结构

```text
C2 整帧结构
├─ B通用包头（固定 24B）
├─ 数据控制区（固定 24B）
├─ IQ数据长度（固定 4B）
├─ IQ数据内容（变长）
└─ B通用包尾（固定 8B）
```

### 4.2 长度关系

1. `B通用包头长度 = 24B`
2. `数据控制区长度 = 24B`
3. `IQ数据长度字段 = 4B`
4. `IQ数据内容长度 = IQ数据字节长度`
5. `B通用包尾长度 = 8B`

因此：

**C2整包长度 = 24 + 24 + 4 + IQ数据字节长度 + 8**

即：

**C2整包长度 = 60 + IQ数据字节长度**

### 4.3 结构说明

数据控制区固定长度为 **24B**，用于描述当前回传 IQ 数据包的波位上下文、采样区间和时间标记。

### 4.7 补充说明

1. C2 独立成帧承载，因此数据控制区中必须保留 `设备编号`。
2. `采样起始`、`采样长度`、`采样点数` 三者均采用“采样点”口径，用于上位机对原始数据进行解释与还原。
3. `PulseIndex` 当前冻结为 `0..2`，采用 0-based 表达，不允许使用 1-based 编号。
4. C2 中的 `时间戳` 采用分解式绝对时间表达：`时 / 分 / 秒 / 毫秒`。
5. 当前版本中，C2 时间戳表示数据打包时刻的绝对时间标签。
6. 当 C2 数据发生分包时，各分包的 `IQ数据字节长度` 分别描述各自包内的数据长度。分包顺序和重组顺序由外层 B 表中的包序号与分包信息统一管理。
7. IQ 数据内容按系统统一固定格式组织，当前版本不在 C 表内重复定义 IQ 数据格式字段。

## 5. 长度关系与解析建议

### 5.1 长度关系

1. C1 整包长度固定为 `84B`。
2. C2 整包长度关系如下：<br>`C2整包长度 = 60 + IQ数据字节长度`

### 5.2 解析建议

1. 接收端必须先完成外层 B 表校验，再依据 `包类型` 选择 C1 或 C2 解析路径。
2. 进入 C1 解析时，建议先检查：<br>`包类型 = 0x03`<br>`状态区长度 = 52`
3. 进入 C2 解析时，建议先检查：<br>`包类型 = 0x04`<br>`控制区标识`<br>`控制区长度 = 24`<br>`IQ数据字节长度` 是否与本包实际剩余长度一致
