# 设计冻结原则汇总

## 1. 适用范围

本文档用于汇总 A 表、B 表、C 表、D 表中已经冻结的设计原则，形成一份独立的原则清单，便于统一查阅、对比和后续新文档沿用。

本文档只做原则提取与归并，不重新定义字段，不替代各表正文中的具体结构、偏移和字段说明。

---

## 2. 通用冻结原则

### 2.1 编码原则

1. 全部文档统一采用**小端编码**。
2. 多字节字段均按小端序解释和传输。

### 2.2 对齐原则

1. 全部文档统一按**4B 自然对齐**设计。
2. 固定结构优先保证 4B 对齐，便于 FPGA 侧 RAM 预分配、状态机解析和 DMA 访问。
3. 若存在变长业务区，为满足 4B 对齐而补齐的零字节，视具体外层封装规则决定是否参与校验。

### 2.3 偏移原则

1. 各文档原则上都采用**统一偏移基准**。
2. 若文档按整帧视角写作，则 `offset` 以整帧起始字节为零点，即以 B 通用包头中的“帧首校验字”所在字节为零点。
3. 若文档为独立业务包体文档，则必须在文首明确声明 `offset` 基准，不允许在同一文档中无说明切换局部偏移基准。

### 2.4 固定结构原则

1. 能采用固定长度表达的结构，优先采用固定长度。
2. 固定长度字段优先显式写出总长度、起止范围和长度关系。
3. 对于需要高效硬件解析的结构，固定长度优先于松散扩展式表达。

### 2.5 去冗余原则

1. 外层通用封装已承担的传输层字段，不在业务层重复设置。
2. 已在慢变装订表中定义的完整模板，不在调度表或回传表中重复复制。
3. 整帧唯一已知的信息，不在后续区段中重复携带，除非该信息对应的是另一种独立包体。

### 2.6 版本冻结原则

1. 每种表结构都应明确当前冻结版本。
2. 版本字段的用途是约束结构解释方式，而不是承载业务语义。
3. 若结构发生实质变化，应通过版本升级明确说明，不允许在旧版本下隐式扩张字段含义。

### 2.7 校验原则

1. 帧首校验字只承担判帧职责，不参与 CRC 计算。
2. CRC 字段自身不参与 CRC 计算。
3. 帧尾校验字若存在，只承担帧结束辅助判定职责，不参与 CRC 计算。
4. CRC 覆盖范围必须在文档中明确写成冻结规则，不允许只用口头描述。

### 2.8 数组索引原则

1. 多脉冲字段统一采用数组化表达。
2. `PulseIndex = 0..2`，统一采用 0-based 表达。
3. `PulseIndex = 0` 表示第一脉冲，`PulseIndex = 1` 表示第二脉冲，`PulseIndex = 2` 表示第三脉冲。
4. 同一索引下的多组数组字段应按“同索引成组解释”原则配套读取。

## 3. A表冻结原则提取

### 3.1 结构冻结原则

1. A 表为**单帧装订表**。
2. A 表采用“专用包头 + 固定业务体 + 整帧 CRC”结构。
3. A 表本版仅保留**整帧 CRC32/IEEE 校验码**，不设置帧尾校验字。

### 3.2 业务边界原则

1. A 表只承载慢变模板参数与资源映射信息。
2. A 表不承载波位级调度逻辑。
3. A 表中的三脉冲模板数量冻结为 `3`。

### 3.3 量化表达原则

1. 量化字段保留在线定义，字段表内直接写明量化单位、换算方式与典型示例。
2. 波形模板字段与存储映射字段必须分开表达，不允许用资源地址替代量化模板语义。

### 3.4 校验与长度原则

1. A 表当前整包长度固定为 `100B`。
2. CRC 覆盖范围冻结为 `offset = 4` 到 `offset = 95`。
3. 解析前应至少完成：帧首校验字、包头长度、整包长度、CRC 四项检查。

---

## 4. B表冻结原则提取

### 4.1 封装职责原则

1. B 表是系统统一传输包格式，作为各类业务表的通用外层封装。
2. B 表只负责判帧、包头识别、整包长度识别、包类型分发、分包组织与整帧校验。
3. B 表不定义具体业务包体字段，不解释业务字段含义，不规定业务层超时、重传、补包、容错策略。

### 4.2 结构冻结原则

1. B 表固定采用“**固定包头 + 变长包体 + 固定包尾**”结构。
2. 包头固定长度为 `24B`。
3. 包尾固定长度为 `8B`。

### 4.3 分包原则

1. B 表保留通用分包能力，以适配大数据量业务包体。
2. 不分包时，分包字段固定为 `1/1`。
3. 分包时，同一业务组必须保持相同包类型、相同包序号、相同分包总数。
4. 接收端应先按 `包类型 + 包序号` 识别业务组，再按 `分包序号` 排列。

### 4.4 包体原则

1. 包体是变长区域，B 表只负责提供封装边界。
2. 包体内部字段建议按 4B 自然对齐设计。
3. 包体末尾为满足对齐而补齐的零字节属于包体的一部分，并参与整帧 CRC 计算。

### 4.5 校验原则

1. B 表整帧校验算法冻结为 **CRC32/IEEE**。
2. CRC 覆盖范围冻结为：从“包头长度”字段开始，到“包体结束”位置为止。
3. 等价整包偏移表达为：`offset = 4` 到 `offset = 整包长度 - 9`。

## 5. C表冻结原则提取

### 5.1 包体划分原则

1. C 表分为两类独立包体：`C1 自然状态回传包体` 与 `C2 波位数据回传包体`。
2. C1 与 C2 不在同一帧中同时出现，而是分别由不同 `包类型` 承载。
3. C1 对应 `包类型 = 0x03`，C2 对应 `包类型 = 0x04`。

### 5.2 自然状态回传原则

1. C1 自然状态回传采用固定结构，不再以“基础结构 + 尾部补丁扩展”作为主表达方式。
2. 当前冻结的 C1 包体长度为 `52B`。
3. 姿态与倾角传感器观测量作为正式区段纳入 C1。
4. 倾角传感器相关量必须成组回传，至少包含 `X`、`Y`、状态字、有效标志字。

### 5.3 波位数据回传原则

1. C2 波位数据回传采用“固定控制区 + 固定长度字段 + 变长数据区”结构。
2. 数据控制区只描述 IQ 数据的解释上下文，不承载外层封装信息。
3. `IQ数据长度` 与 `IQ数据内容` 的表达固定化，不再使用“同相正交”术语。

### 5.4 时间与计数原则

1. `数据时间标记` 冻结为本地自由运行探测周期计数。
2. `数据时间标记` 不直接表示 UTC 绝对时间。
3. `波位内积累序号` 是回传结果中的执行序号，不等同于调度表中的计划积累数量。

### 5.5 姿态观测原则

1. 航向角采用 `0.001°/LSB` 量化。
2. 倾角传感器 `X/Y` 采用 `0.001°/LSB` 量化。
3. 倾角传感器状态字与有效标志字必须与数值字段成组解释，不得孤立使用数值字段。

## 6. D表冻结原则提取

### 6.1 调度结构原则

1. D 表采用“表头 + 固定波位项”结构。
2. D 表头当前冻结为 `12B`。
3. 波位项当前冻结为 `36B` 固定结构。
4. 固定波位项设计优先服务于 FPGA 侧 RAM 预分配、状态机解析和 DMA 访问。

### 6.2 去冗余原则

1. D 表不重复定义 B 表已有的传输层字段。
2. D 表不重复复制 A 表中已装订的完整波形定义。
3. D 表头不再重复设置“表标识”和“表头长度”字段。
4. 频点代码采用索引引用方式，而不是在波位项中重复下发完整频点数组。

### 6.3 调度语义原则

1. `波位内积累个数` 表示当前波位计划执行的积累数量。
2. `波位内积累个数` 不等同于 C 表中的 `波位内积累序号`。
3. 时间戳字段仅定义为 `时/分/秒/ms` 的分解式时间戳，不附加更强的软件调度语义。

### 6.4 生效原则

1. D 表统一采用“探测周期边界下一周期生效”规则。
2. 校验通过后先写入 Shadow Buffer。
3. 待下一探测周期边界到达时切换为 Active Buffer。
4. 切换后从新表第 0 个波位项开始执行。

### 6.5 倾角补偿原则

1. D 表允许携带波位级姿态补偿输入参数。
2. 倾角补偿字段用于波位控制时考虑倾角补偿。
3. 倾角补偿字段通常来源于自然状态回传中的倾角传感器观测结果，经上位机处理后再写入 D 表。
4. D 表中的倾角补偿字段是控制输入值，不是传感器原始观测回传副本。

## 7. 跨表统一约束

### 7.1 包类型统一约束

1. `0x01` 对应 A 表一次性装订包。
2. `0x02` 对应 D 表波位控制包。
3. `0x03` 对应 C1自然状态回传包。
4. `0x04` 对应 C2波位数据回传包。

### 7.2 索引统一约束

1. `PulseIndex = 0..2` 在 A/C/D 中保持一致。
2. `PulseIndex` 统一采用 0-based 表达。
3. 所有三脉冲数组字段都应按相同索引解释。

### 7.3 量化统一约束

1. `方位角/俯仰角` 在波位控制相关场景统一采用 `0.0025°/LSB`。
2. `MGC增益代码` 统一采用 `0.5 dB/LSB`。
3. `采样点数/采样起始/采样长度` 统一采用“采样点”口径。
4. `航向角` 与 `倾角传感器X/Y` 在姿态观测相关场景统一采用 `0.001°/LSB`。

### 7.4 波形组统一约束

1. A 表负责装订波形组。
2. D 表通过 `波形组编号` 引用对应装订结果。
3. D 表中 `波形组编号` 与 A 表表头中的 `波形组编号` 保持一致。

---

## 8. 使用建议

1. 后续新增文档时，应先继承本文件中的通用冻结原则，再补充该文档专属边界。
2. 若某项原则发生跨表级别变化，应优先更新本汇总文档，再同步更新各表正文。
3. 若某项规则仅影响单表实现，应只在对应表正文中冻结，不必写入本汇总文档。
