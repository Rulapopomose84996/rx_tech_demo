# A表 一次性装订表文档

## 1. 适用范围

A 表用于服务器向前端设备一次性下装**慢变模板参数**与**资源映射信息**。本表面向以下内容：

1. 三脉冲波形模板参数。
2. 接收与采样链路模板参数。
3. 慢变资源区与保护策略参数。
4. 波形资源与滤波资源的存储映射参数。

本表是**单帧装订表**，整帧长度远小于 `1500B`，因此本版仅保留**整帧 CRC32/IEEE 校验码**，不设置帧尾校验字。

---

## 2. 完整包结构

```text
┌──────────────────────────────────────────┐
│ A 表表头（固定 16B）                      │
│   帧首校验字 55AAAA55 / 包类型 / 波形组编号 │
├──────────────────────────────────────────┤
│ A 表业务区（固定 80B）                    │
│  ├─ 三脉冲波形模板区（24B）              │
│  ├─ 接收与采样模板区（16B）              │
│  ├─ 资源与保护配置区（8B）               │
│  └─ 存储映射区（32B）                    │
├──────────────────────────────────────────┤
│ 整帧 CRC32/IEEE（固定 4B）                │
└──────────────────────────────────────────┘
```

**整包长度 = 16B + 80B + 4B = 100B**

---

## 3. A 表表头

### 3.1 结构说明

A 表表头固定长度为 **16B**，用于完成判帧、版本识别、包类型识别、整包长度识别和波形组识别。

本版删除以下字段：

1. 包序号
2. 分包总数
3. 分包序号
4. 预留字段 1
5. 预留字段 2

同时将原业务控制区中的 `波形组编号` 迁移至 A 表表头内维护。

### 3.2 字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 0 | 帧首校验字 | `uint32` | 4B | 固定值 `0x55AAAA55` | 无 | **用于快速定位 A 表帧首。**<br>发送端固定写入 `0x55AAAA55`。<br>接收端若读取值不匹配，应立即判定当前字节流不是合法 A 表起始，不进入后续字段解析。 |
| 4 | 表头长度 | `uint16` | 2B | 固定值 `16` | `1B/LSB` | **表示 A 表表头总长度。**<br>本版固定为 `16`。<br>接收端可先检查该字段是否为 `16`，再决定后续偏移是否按本版格式解释。 |
| 6 | 协议版本 | `uint8` | 1B | 当前冻结 `0x10` | 高 4 位主版本<br>低 4 位次版本 | **表示 A 表当前协议版本。**<br>本版取值固定为 `0x10`。<br>示例：`0x10` 表示主版本 `1`、次版本 `0`。<br>若主版本不匹配，接收端不得按本版字段表继续解释。 |
| 7 | 包类型 | `uint8` | 1B | 固定值 `0x01` | 无量纲编码 | **表示当前帧为 A 表一次性装订包。**<br>本字段用于包类型分发。<br>当前冻结值为 `0x01`。 |
| 8 | 整包长度 | `uint32` | 4B | 固定值 `100` | `1B/LSB` | **表示整帧总长度。**<br>本版 A 表固定为 `100B`。<br>接收端应校验实际接收长度是否与该字段一致；不一致时应拒绝本帧。 |
| 12 | 波形组编号 | `uint16` | 2B | `0..65535` | 无 | **标识本次装订所属的波形模板组。**<br>该编号用于把本帧中的三脉冲模板、资源区参数和存储映射视为同一组慢变装订内容。<br>示例：`波形组编号 = 12` 表示当前帧装订的是第 12 组模板。 |
| 14 | 预留字段 | `uint16` | 2B | 当前冻结 `0x0000` | 无 | **预留扩展字段。**<br>当前版本发送端必须写 `0`。 |

### 3.3 补充说明

1. `offset = 0..15` 全部属于 A 表表头范围，后续业务字段一律从 `offset = 16` 开始。
2. `表头长度` 与 `整包长度` 必须同时满足：`表头长度 = 16`、`整包长度 = 100`。任一条件不成立，均应拒绝装订。
3. `波形组编号` 前移到包头后，接收端在进入业务体前即可确认本帧所属模板组，便于做装订前筛选和日志归类。
4. `预留字段` 当前版本必须写 `0`，不得赋予业务语义。

## 4. A 表业务区

### 4.1 结构说明

A 表业务区固定长度为 **80B**，从 `offset = 16` 开始，到 `offset = 95` 结束。

业务区由 4 个连续区域组成：

1. 三脉冲波形模板区：`offset = 16..39`，固定 `24B`
2. 接收与采样模板区：`offset = 40..55`，固定 `16B`
3. 资源与保护配置区：`offset = 56..63`，固定 `8B`
4. 存储映射区：`offset = 64..95`，固定 `32B`

### 4.2 补充说明

1. 本版已删除独立的业务控制区，不再单独设置 `业务区长度`、`业务区版本`、`三脉冲模板数量`、`预留字段` 等控制字段。
2. 业务区长度由表头中的 `整包长度` 和固定结构共同确定，无需再重复携带。
3. 三脉冲模板数量在本版中直接冻结为 `3`，通过字段数组长度体现，不再重复设置数量字段。

## 5. 三脉冲波形模板区

### 5.1 结构说明

三脉冲波形模板区固定长度为 **24B**，按“同类属性并排、三脉冲数组展开”的方式组织。

本区所有字段均采用统一的数组索引表达：

- `字段名[0]`：第一脉冲
- `字段名[1]`：第二脉冲
- `字段名[2]`：第三脉冲

例如：

- `脉冲体制代码[0]` 表示第一脉冲采用的体制
- `脉冲工作频点代码[2]` 表示第三脉冲采用的频点代码
- `脉冲采样点数[1]` 表示第二脉冲默认采样点数

### 5.2 字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 16 | 脉冲体制代码[3] | `uint8[3]` | 3B | 每项 `0x00..0xFF` | 无量纲编码 | **标识三路脉冲对应的波形体制类别。**<br>推荐冻结值：`0x01`=短脉冲模板，`0x02`=中脉冲模板，`0x03`=长脉冲模板，`0xFF`=该脉冲位禁用。<br>示例：`[0x01, 0x02, 0x03]` 表示第一、二、三脉冲分别装订为短/中/长模板。 |
| 19 | 脉冲工作频点代码[3] | `uint8[3]` | 3B | 每项 `0..100` | `10 MHz/LSB` | **标识三路脉冲的工作频点代码。**<br>换算方式：`实际频点 = 15.5 GHz + 代码值 × 10 MHz`。<br>示例：代码 `0` 表示 `15.5 GHz`，代码 `25` 表示 `15.75 GHz`，代码 `100` 表示 `16.5 GHz`。 |
| 22 | 脉冲信号带宽代码[3] | `uint8[3]` | 3B | 每项 `0..255` | `0.5 MHz/LSB` | **标识三路脉冲模板的信号带宽。**<br>换算方式：`实际带宽 = 代码值 × 0.5 MHz`。<br>示例：代码 `40` 表示 `20 MHz`，代码 `80` 表示 `40 MHz`。 |
| 25 | 脉冲发射脉宽代码[3] | `uint8[3]` | 3B | 每项 `0..255` | `0.5 μs/LSB` | **标识三路脉冲的发射脉宽。**<br>换算方式：`实际脉宽 = 代码值 × 0.5 μs`。<br>示例：代码 `2` 表示 `1.0 μs`，代码 `10` 表示 `5.0 μs`，代码 `42` 表示 `21.0 μs`。 |
| 28 | 脉冲重复间隔代码[3] | `uint8[3]` | 3B | 每项 `0..255` | `1 μs/LSB` | **标识单个探测周期内三路脉冲的默认重复间隔。**<br>换算方式：`实际间隔 = 代码值 × 1 μs`。<br>该字段用于描述对应脉冲模板的默认时间组织。<br>示例：代码 `5` 表示 `5 μs`。 |
| 31 | 脉冲MGC增益代码[3] | `uint8[3]` | 3B | 每项 `0..125` | `0.5 dB/LSB` | **标识三路脉冲对应接收链路的默认 MGC 增益。**<br>换算方式：`实际增益 = 代码值 × 0.5 dB`。<br>示例：代码 `40` 表示 `20 dB`，代码 `80` 表示 `40 dB`。 |
| 34 | 脉冲采样点数[3] | `uint16[3]` | 6B | 每项 `1..65535` | `1 点/LSB` | **标识三路脉冲默认采样点数。**<br>该字段直接表示采样点数量，不是时间长度。<br>示例：`1024` 表示该脉冲默认采样 `1024` 点；若结合抽取倍率为 `2`，则离线解释时应按 2 倍抽取后的点间隔理解。 |

### 5.3 补充说明

1. 本区总字节数计算如下：`6 个 uint8[3] 字段 = 18B`，`1 个 uint16[3] 字段 = 6B`，合计 `24B`。
2. 三脉冲数组采用**同索引成组解释**原则。也就是说，`PulseIndex = 1` 时，应同时读取：<br>`脉冲体制代码[1]`<br>`脉冲工作频点代码[1]`<br>`脉冲信号带宽代码[1]`<br>`脉冲发射脉宽代码[1]`<br>`脉冲重复间隔代码[1]`<br>`脉冲MGC增益代码[1]`<br>`脉冲采样点数[1]`
3. 推荐的三脉冲典型装订示例如下：<br>`脉冲体制代码 = [0x01, 0x02, 0x03]`<br>`脉冲工作频点代码 = [10, 12, 15]`<br>`脉冲信号带宽代码 = [20, 40, 80]`<br>`脉冲发射脉宽代码 = [2, 10, 42]`<br>`脉冲重复间隔代码 = [5, 21, 50]`<br>`脉冲MGC增益代码 = [40, 40, 36]`<br>`脉冲采样点数 = [512, 1024, 2048]`
4. 若某一路脉冲模板未启用，推荐同时设置：<br>`脉冲体制代码[i] = 0xFF`<br>其余同索引量化字段写 `0`<br>对应存储映射字段中的地址、长度写 `0`

## 6. 接收与采样模板区

### 6.1 结构说明

接收与采样模板区固定长度为 **16B**，用于描述采样门、通道输出、数据组织、触发方式等慢变接收参数。

本区字段不区分三脉冲索引，而是作为本次装订统一生效的模板配置。

### 6.2 字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 40 | 采样起始点 | `uint16` | 2B | `0..65535` | `1 点/LSB` | **标识统一采样窗口的起始采样点。**<br>该值表示从接收数据流第几个采样点开始取数。<br>示例：`256` 表示从第 `256` 点开始采样。 |
| 42 | 采样长度 | `uint16` | 2B | `1..65535` | `1 点/LSB` | **标识统一采样窗口的长度。**<br>该值表示连续采样的点数总量。<br>示例：`2048` 表示采样窗口长度为 `2048` 点。 |
| 44 | 采样波门起始时延代码 | `uint16` | 2B | `0..65535` | `0.01 μs/LSB` | **标识采样波门相对探测周期基准点的起始时延。**<br>换算方式：`实际时延 = 代码值 × 0.01 μs`。<br>示例：代码 `290` 表示 `2.90 μs`。 |

| 52 | 滤波器组编号 | `uint8` | 1B | `0..255` | 无量纲编码 | **标识当前模板默认调用的滤波器组编号。**<br>该字段与资源区编号、滤波器组版本、滤波系数存储地址共同确定实际滤波资源。<br>示例：`3` 表示调用第 3 组滤波器。 |
| 53 | 自动增益控制模式 | `uint8` | 1B | `0x00..0xFF` | 无量纲编码 | **标识增益控制模式。**<br>推荐取值：`0x00`=仅使用手动增益代码，`0x01`=开机装订后自动接管，`0x02`=手动增益起始后自动微调，`0xFF`=禁用配置。 |
| 54 | 接收窗口选择字 | `uint8` | 1B | `0x00..0xFF` | 位定义 | **标识默认启用的接收窗口组合。**<br>推荐位定义：`Bit0`=窗口 R1 使能，`Bit1`=窗口 R2 使能，`Bit2`=窗口 R3 使能，`Bit3`=三窗口公共波门，`Bit4`=允许窗口独立时延，`Bit5..7` 保留。<br>示例：`0x07` 表示 R1/R2/R3 三窗口全部启用。 |
| 55 | 触发模式代码 | `uint8` | 1B | `0x00..0xFF` | 无量纲编码 | **标识接收链路触发方式。**<br>推荐取值：`0x00`=内部周期触发，`0x01`=外部硬触发，`0x02`=PPS 对齐触发，`0x03`=调试单次触发。 |

### 6.3 补充说明

1. `采样起始点` 与 `采样长度` 共同确定采样点区间：<br>`采样区间 = [采样起始点, 采样起始点 + 采样长度 - 1]`
2. `采样波门起始时延代码` 与 `采样波门持续时长代码` 用于给出**时间域**上的门控配置；`采样起始点` 与 `采样长度` 用于给出**离散采样点域**上的取数范围。二者应保持配置一致，不能出现时间窗覆盖 1000 点、而点数窗只允许 256 点的矛盾配置。
3. `通道使能字` 推荐采用“按位开关、未定义位清零”的使用方式。若仅需要基础双通道输出，可设置为：<br>`Bit0 = 1`、`Bit1 = 1`、其余位为 `0`，对应数值 `0x0003`。
4. `抽取倍率` 会影响离线换算。若原始采样时钟为 `Fs`，则抽取后的等效采样时钟为 `Fs / 抽取倍率`。
5. `接收窗口选择字` 与 `触发模式代码` 应由发送端成对考虑。典型安全配置示例如下：<br>`接收窗口选择字 = 0x07`<br>`触发模式代码 = 0x00`<br>表示三窗口均启用、由内部周期触发。

## 7. 资源与保护配置区

### 7.1 结构说明

资源与保护配置区固定长度为 **8B**，用于冻结当前模板所绑定的资源区、滤波器版本和装订保护策略。

### 7.2 字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 56 | 资源区编号 | `uint16` | 2B | `0..65535` | 无 | **标识本次 A 表装订所使用的逻辑资源区编号。**<br>波形资源地址和滤波资源地址均在该资源区内解释。<br>示例：`2` 表示资源区 2。 |
| 58 | 滤波器组版本 | `uint16` | 2B | `0..65535` | 无 | **标识当前滤波器组的版本号。**<br>该字段用于确认滤波资源是否与模板一致。<br>示例：`202` 可表示滤波资源版本 2.02。 |
| 60 | 故障保护方式 | `uint8` | 1B | `0x00..0xFF` | 枚举编码 | **标识装订失败或运行校验失败时的保护动作。**<br>推荐取值：`0x00`=拒绝装订并保持原配置，`0x01`=停止发射但保持接收，`0x02`=停止收发并进入保护态，`0x03`=切回安全模板。 |
| 61 | 校验策略代码 | `uint8` | 1B | `0x00..0xFF` | 枚举编码 | **标识本次装订执行的业务校验等级。**<br>推荐取值：`0x00`=仅检查长度与 CRC，`0x01`=检查长度/CRC/地址范围，`0x02`=检查长度/CRC/地址范围并执行回读确认。 |
| 62 | 预留字段[2] | `uint8[2]` | 2B | 固定值 `0x00 0x00` | 无 | **预留扩展字段。**<br>当前版本发送端必须写 `0`。<br>接收端不得将其解释为业务含义。 |

### 7.3 补充说明

1. `资源区编号` 是存储映射区各地址字段的**解释前提**。同一个地址值落在不同资源区编号下，代表的实际物理资源位置可能完全不同。
2. `滤波器组版本` 建议与生产、烧录、回读工具保持同一版本命名规则，避免“模板编号正确但滤波资源版本错误”的隐患。
3. `故障保护方式` 与 `校验策略代码` 应成对配置。<br>推荐安全配置：`故障保护方式 = 0x00`、`校验策略代码 = 0x02`。<br>含义为：一旦回读确认失败，则拒绝本次装订并保持原配置。
4. `预留字段[2]` 仅用于后续版本扩展，本版不得承载“临时实验配置”或“现场口头约定配置”。

## 8. 存储映射区

### 8.1 结构说明

存储映射区固定长度为 **32B**，用于给出三脉冲波形资源和滤波资源在目标资源区中的存储位置与字节长度。

本区所有地址和长度均采用**字节口径**，不采用采样点口径。

### 8.2 字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 64 | 脉冲波形存储基地址[3] | `uint32[3]` | 12B | 每项 `0..0xFFFFFFFF` | `1B/LSB` | **标识三路脉冲波形资源在当前资源区内的起始字节偏移。**<br>数组索引与三脉冲模板区完全一致。<br>示例：`[0x00001000, 0x00002000, 0x00004000]` 表示三路脉冲波形分别从 `0x1000`、`0x2000`、`0x4000` 偏移开始存放。 |
| 76 | 滤波系数基地址 | `uint32` | 4B | `0..0xFFFFFFFF` | `1B/LSB` | **标识当前滤波器系数资源在当前资源区内的起始字节偏移。**<br>示例：`0x00008000` 表示滤波系数从资源区偏移 `0x8000` 开始。 |
| 80 | 脉冲波形字节长度[3] | `uint32[3]` | 12B | 每项 `0..0xFFFFFFFF` | `1B/LSB` | **标识三路脉冲波形资源的字节长度。**<br>该字段描述的是波形资源文件或波形块的长度，不是脉宽，不是采样点数。<br>示例：`[4096, 8192, 12288]` 表示三路脉冲波形资源长度分别为 `4KB`、`8KB`、`12KB`。 |
| 92 | 滤波系数字节长度 | `uint32` | 4B | `0..0xFFFFFFFF` | `1B/LSB` | **标识滤波器系数资源总字节长度。**<br>示例：`2048` 表示滤波系数资源长度为 `2048B`。 |

### 8.3 补充说明

1. 存储映射区采用“地址 + 长度”成对描述方式，避免使用额外的存储区号字段，从而保持结构紧凑。
2. 三脉冲波形资源的解释方法如下：<br>`PulseIndex = 0` 时，读取 `脉冲波形存储基地址[0]` 与 `脉冲波形字节长度[0]`<br>`PulseIndex = 1` 时，读取 `脉冲波形存储基地址[1]` 与 `脉冲波形字节长度[1]`<br>`PulseIndex = 2` 时，读取 `脉冲波形存储基地址[2]` 与 `脉冲波形字节长度[2]`
3. 典型装订示例如下：<br>`资源区编号 = 2`<br>`脉冲波形存储基地址 = [0x1000, 0x3000, 0x7000]`<br>`脉冲波形字节长度 = [0x0800, 0x1000, 0x1800]`<br>`滤波系数基地址 = 0xA000`<br>`滤波系数字节长度 = 0x0400`<br>含义是：在资源区 2 内，三路波形分别占用 `0x1000~0x17FF`、`0x3000~0x3FFF`、`0x7000~0x87FF`，滤波系数占用 `0xA000~0xA3FF`。
4. 发送端应保证地址与长度组合满足以下约束：<br>`字节长度 > 0` 时，对应基地址必须有效；<br>`字节长度 = 0` 时，对应基地址应同时写 `0`；<br>同一资源区内不同资源块不得出现重叠。
5. 若三路脉冲共用同一资源文件，允许把多个 `脉冲波形存储基地址[i]` 指向同一地址，但此时各自的 `脉冲体制代码[i]`、频点、带宽、脉宽等量化字段也应保持一致，避免形成“同资源、不同模板”的歧义配置。

## 9. 整帧 CRC32/IEEE

### 9.1 字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 96 | 整帧校验码 | `uint32` | 4B | `0..0xFFFFFFFF` | CRC32/IEEE 结果值 | **表示当前整帧的 CRC32/IEEE 校验结果。**<br>该字段是本版 A 表唯一帧级校验字段。<br>本版不设置帧尾校验字。 |

### 9.2 CRC 计算规则

CRC 计算范围冻结如下：

1. 起始偏移：`offset = 4`
2. 结束偏移：`offset = 95`
3. 不包含 `offset = 0..3` 的帧首校验字
4. 不包含 `offset = 96..99` 的 CRC 字段自身

### 9.3 CRC 参数

| 参数项 | 冻结值 |
| -- | -- |
| CRC 名称 | `CRC-32/IEEE` |
| 生成多项式 | `0x04C11DB7` |
| 初始值 | `0xFFFFFFFF` |
| 输入反射 | `true` |
| 输出反射 | `true` |
| 最终异或值 | `0xFFFFFFFF` |
| 结果写入字节序 | 小端写入 |

### 9.4 补充说明

1. CRC 输入顺序按报文实际发送字节顺序处理，不额外调整字序。
2. 由于本表总长度固定为 `100B`，因此可以直接按 `offset = 4..95` 的固定区间计算 CRC，不需要额外查找帧尾。
3. 接收端完成字段解析前，至少应先完成以下检查：<br>帧首校验字正确<br>包头长度正确<br>整包长度正确<br>CRC 正确
4. 本版删除帧尾校验字后，整帧结束位置仅通过 `整包长度 = 100` 与底层接收长度共同判定。

## 10. 装订与校验建议

1. 发送端建议按以下顺序生成 A 表：<br>先填写表头字段<br>再填写三脉冲模板区、接收与采样模板区、资源与保护配置区、存储映射区<br>最后计算并写入 `offset = 96` 的整帧校验码。
2. 接收端建议按以下顺序校验 A 表：<br>先校验帧首校验字<br>再校验表头长度与整包长度<br>再校验 CRC32/IEEE<br>CRC 通过后再做业务字段合法性检查。
3. 业务字段合法性检查至少应覆盖以下内容：<br>三脉冲数组字段是否长度一致<br>地址与长度是否成对有效<br>采样窗口与波门配置是否存在明显矛盾<br>枚举字段是否落在已定义范围内。
4. 若采用严格装订策略，推荐把 `校验策略代码` 设为 `0x02`，并在装订完成后执行一次资源区回读确认。
