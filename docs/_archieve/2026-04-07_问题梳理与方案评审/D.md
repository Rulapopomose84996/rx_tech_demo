# D表 波位安排表文档

## 1. 适用范围

D 表用于定义前端执行侧的波位级安排信息，用于描述一次安排批次内各波位的执行顺序、工作方式、指向参数、三脉冲频点组合索引、三脉冲增益参数、采样参数、波位拼接参数、波位内积累参数以及波位级倾角补偿输入参数。

D 表作为业务包体使用，外层统一采用 B 表通用封装格式承载。

---

## 2. 整体封装结构

```text
整帧结构
├─ B通用包头（固定 24B）
├─ D表头（固定 12B）
├─ 波位项[0]（固定 36B）
├─ 波位项[1]（固定 36B）
├─ ~.
├─ 波位项[N-1]（固定 36B）
└─ B通用包尾（固定 8B）
```

### 2.1 长度关系

1. `B通用包头长度 = 24B`
2. `D表头长度 = 12B`
3. `波位项长度 = 36B`
4. `B通用包尾长度 = 8B`

因此整包长度可写为：

**整包长度 = 24 + 12 + N × 36 + 8**

即：

**整包长度 = 44 + N × 36**

其中 `N` 为本表波位数。

---

## 3. B通用包头与包尾位置说明

### 3.1 B通用包头字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 说明 |
| -- | -- | -- | --: | -- |
| 0 | 帧首校验字 | `uint32` | 4B | 固定值 `0x55AAAA55`，用于判帧起始。 |
| 4 | 包头长度 | `uint16` | 2B | 固定值 `24`。 |
| 6 | 协议版本 | `uint8` | 1B | 当前冻结 `0x10`。 |
| 7 | 包类型 | `uint8` | 1B |波位控制表。 |
| 8 | 整包长度 | `uint32` | 4B | 当前整帧总长度。 |


### 3.2 B通用包尾字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 说明 |
| -- | -- | -- | --: | -- |
| 包尾起始 + 0 | 整帧校验码 | `uint32` | 4B | CRC32/IEEE，覆盖范围由 B 通用格式定义。 |
| 包尾起始 + 4 | 帧尾校验字 | `uint32` | 4B | 固定值 `0xAA5555AA`。 |

### 3.3 补充说明

1. D 表头从 `offset = 24` 开始，原因是其前面固定存在 `24B` 的 B 通用包头。
2. 第 0 个波位项从 `offset = 36` 开始，原因是其前面固定存在 `24B` B 通用包头和 `12B` D 表头。
3. B 通用包尾不单独参与 D 表业务字段解释，但必须保留在本文档中，便于整体偏移理解。

## 4. D表头

### 4.1 结构说明

D 表头固定长度为 **12B**，用于描述当前 D 表业务体的结构信息和波位项组织方式。

### 4.2 字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 26 | 波位基准个数 | `uint16` | 2B | `1~65535` | `1 项/LSB` | **表示当前 D 表中实际包含的波位项数量。**<br>接收端必须依据该字段解析后续波位项个数。 |
| 28 | 波形组编号 | `uint16` | 2B | `0~65535` | 无 | **引用 A 表中已装订的波形组编号。**<br>当前字段与 A 表表头中的 `波形组编号` 保持一致。<br>D 表中的频点索引、脉冲参数索引等均在该波形组上下文中解释。 |

### 4.3 补充说明

1. `本表波位数 = N` 时，整份 D 表业务体长度应满足：`12 + N × 36`。
2. `波形组编号` 用于引用 A 表中已装订的波形模板资源，因此 D 表只负责安排与调用，不重复复制慢变波形定义。
3. `表结构版本 = 0x0102` 表示当前版本已包含波位级倾角补偿输入字段。
4. 新下发 D 表统一采用“探测周期边界下一周期生效”规则：<br>校验通过后先写入 Shadow Buffer，待下一探测周期边界到达时切换为 Active Buffer，并从新表第 0 个波位项开始执行。

---

## 5. 波位项

### 5.1 结构说明

每个波位项固定长度为 **36B**。

三脉冲相关参数统一采用数组化结构，索引体系统一冻结为：

- `PulseIndex = 0`：第一脉冲
- `PulseIndex = 1`：第二脉冲
- `PulseIndex = 2`：第三脉冲

### 5.2 字段表

以下字段表以**第 0 个波位项**为例列出整帧偏移。
当 `BeamIndex = k` 时，该波位项内每个字段的实际偏移为：

**实际偏移 = 基准偏移 + k × 36**

| 偏移（波位项0） | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 36 | 波位号 | `uint16` | 2B | `0~65535` | 无 | **标识当前安排项所属波位编号。**<br>建议在同一张 D 表内保持唯一。 |
| 38 | 工作方式字 | `uint8` | 1B | `0x00~0xFF` | 位定义 | **表示当前波位的工作方式组合。**<br>包括工作类别、频点方式、搜跟方式、展宽方式、T/R 辐射使能。<br>详细位定义见第 7 节。 |
| 40 | 方位角 | `int16` | 2B | `-26000~26000` 推荐 | `0.0025°/LSB` | **表示当前波位方位角。**<br>换算方式：`实际角度 = 编码值 × 0.0025°`。<br>示例：`10000` 表示 `25.000°`。 |
| 42 | 俯仰角 | `int16` | 2B | `-26000~26000` 推荐 | `0.0025°/LSB` | **表示当前波位俯仰角。**<br>换算方式：`实际角度 = 编码值 × 0.0025°`。 |
| 55 | 脉冲MGC增益代码[3] | `uint8[3]` | 3B | 每项 `0~125` | `0.5 dB/LSB` | **表示三脉冲 MGC 增益代码。**<br>换算方式：`实际增益 = 编码值 × 0.5 dB`。<br>`PulseIndex = 0~2` 分别对应三脉冲。 |
| 58 | **四组频率代码** | `uint8` | 1B | `0~59` | 无 | **表示四组频率代码。** |
| 60 | 时间戳-时 | `uint8` | 1B | `0~23` | `1 h/LSB` | **表示时间戳中的“时”。** |
| 61 | 时间戳-分 | `uint8` | 1B | `0~59` | `1 min/LSB` | **表示时间戳中的“分”。** |
| 62 | 时间戳-秒 | `uint8` | 1B | `0~59` | `1 s/LSB` | **表示时间戳中的“秒”。** |
| 63 | 时间戳-毫秒 | `uint16` | 2B | `0~999` | `1 ms/LSB` | **表示时间戳中的“毫秒”。**<br>采用 `uint16` 小端存储。 |
| 65 | 倾角补偿X | `int16` | 2B | `-32768~32767` | `0.001°/LSB` | **表示当前波位采用的 X 轴倾角补偿输入值。**<br>该值通常来源于自然状态回传中的倾角传感器X，经上位机处理后下发。<br>换算方式：`实际补偿角 = 编码值 × 0.001°`。 |
| 67 | 倾角补偿Y | `int16` | 2B | `-32768~32767` | `0.001°/LSB` | **表示当前波位采用的 Y 轴倾角补偿输入值。**<br>该值通常来源于自然状态回传中的倾角传感器Y，经上位机处理后下发。 |
| 69 | 倾角补偿控制字 | `uint8` | 1B | `0x00~0xFF` | 位定义 | **表示当前波位的倾角补偿控制方式。**<br>推荐位定义：`Bit0`=倾角补偿使能，`Bit1`=使用 X/Y 补偿输入，`Bit2`=允许执行侧二次修正，`Bit3`=补偿值来自最新自然状态，`Bit4~7` 预留。 |
| 70 | 预留字段[2] | `uint8[2]` | 2B | 当前冻结 `0x00 0x00` | 无 | **预留扩展字段。**<br>当前版本发送端必须写 `0`。 |

### 5.3 补充说明

1. 第 `k` 个波位项起始偏移计算公式：<br>`波位项[k]起始偏移 = 36 + k × 36`
2. `脉冲采样点数[3]` 与 `脉冲MGC增益代码[3]` 都采用数组化组织，统一使用 `PulseIndex = 0~2`。
3. `三脉冲频点组合索引` 引用的是“三脉冲频点组合”，不是单一频点。
4. D 表中的倾角补偿字段用于波位级控制补偿输入，不表示传感器原始观测数据，也不替代自然状态回传中的姿态观测职责。

## 6. 工作方式字位定义

### 6.1 字段长度

`工作方式字` 长度固定为 **1B**。

### 6.2 位定义

| 位段 | 含义 | 定义 |
| -- | -- | -- |
| `D1-D0` | 工作类别 | `00` 工作，`01` 校准，`10` 测试，`11` 预留 |
| `D2` | **频点方式** | `0` 单频，`1` 多频 |
| `D4-D3` | 搜跟方式 | `00` 搜索，`01` 跟踪，`10` 搜跟混合，`11` 预留 |
| `D6-D5` | 展宽方式 | `00` 不展宽，`01` 接收展宽，`10` 发射展宽，`11` 收发展宽 |
| `D7` | T/R 辐射使能 | `0` 关闭，`1` 开启 |




### 6.3 补充说明

1. `工作方式字` 是一个组合控制字，接收端必须按位段拆解解释。
2. 对未定义位组合，当前版本应视为非法或保留组合，不建议发送端使用。

---

## 7. 三脉冲频点组合索引定义

### 7.1 字段作用

`三脉冲频点组合索引` 用于引用系统内预置的三脉冲频点代码数组，而不在波位项中分别写入三组频点代码。

### 7.2 建议解释方式

建议系统维护一张**频点代码数组表**，每一项包含：

1. 第 1 脉冲频点代码
2. 第 2 脉冲频点代码
3. 第 3 脉冲频点代码

D 表只下发索引号，执行侧根据索引号查表取得对应三脉冲频点代码。

### 7.3 补充说明

1. `三脉冲频点组合索引 = 0` 表示引用第 0 组频点代码数组。
2. `三脉冲频点组合索引 = 1` 表示引用第 1 组频点代码数组。
3. 频点代码数组的解释上下文应与 D 表头中的 `波形组编号` 保持一致。

---

## 8. 长度关系与解析建议

### 8.1 长度关系

1. D 表头当前固定长度为 `12B`，其起始偏移为 `24`，结束偏移为 `35`。
2. 第 0 个波位项起始偏移为 `36`。
3. 单个波位项长度固定为 `36B`。
4. 当本表波位数为 `N` 时：<br>`D表业务体长度 = 12 + N × 36`
5. 整包长度关系如下：<br>`整包长度 = 24 + 12 + N × 36 + 8`

### 8.2 解析建议

1. 接收端在进入 D 表解析前，应先完成外层 B 表合法性校验。
2. 进入 D 表解析时，建议先检查：<br>`波位项长度 = 36`<br>`本表波位数`<br>`表结构版本`
3. 解析第 `k` 个波位项时，建议按如下公式定位：<br>`offset = 36 + k × 36`
4. 对倾角补偿输入字段，建议先检查 `倾角补偿控制字`，再决定是否采用 `倾角补偿X/Y` 参与当前波位补偿计算。
