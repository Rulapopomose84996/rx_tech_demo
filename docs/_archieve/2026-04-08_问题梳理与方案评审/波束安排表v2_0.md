# 波束安排表 v2.0

## 1. 适用范围

本文档用于定义前端执行侧的波位级安排信息，描述一次下发批次内各波位的执行顺序、工作模式、校准模式、检查模式、波束指向、增益控制、倾角补偿输入、校准后配相数据、跟踪距离、噪声门限和时间戳。

本表作为 D 表业务包体使用，采用统一整帧格式承载，`包类型` 固定为 `0x02`。

本文档采用以下记号：

1. `N` 表示 `波位中CPI计数`。
2. `k` 表示波位项索引，取值范围为 `0~N-1`。
3. 第 `k` 个波位项的实际起始偏移为 `20 + 136×k`。

## 2. 整帧结构与长度关系

```text
整帧结构
├─ 固定区（固定 20 byte）
├─ 波位项[0]（固定 136 byte）
├─ 波位项[1]（固定 136 byte）
├─ ...
├─ 波位项[N-1]（固定 136 byte）
└─ 结束校验区（固定 8 byte）
```

长度关系如下：

1. `固定区长度 = 20`
2. `波位项长度 = 136`
3. `结束校验区长度 = 8`

因此整包长度为：

`整包长度 = 20 + 136×N + 8`

即：

`整包长度 = 28 + 136×N`

## 4. 波位项数组字段设计说明

### 4.1 主波束阵面坐标 `mainBeamCoord[2]`

1. 类型为 `int32[2]`，总字节数为 `8`。
2. 存储顺序为行内连续存储，先写方位角，再写俯仰角。
3. 索引定义如下：`mainBeamCoord[0]` 表示主方位角，`mainBeamCoord[1]` 表示主俯仰角。
4. 量化统一为 `0.001deg`，线上值 `1000` 表示 `1.000deg`。
5. 该字段表示当前波位的默认阵面指向。当四频点指向未启用时，默认指向同时适用于当前波位内所有启用频点。

### 4.2 四频点波束阵面坐标 `freqBeamCoord[4][2]`

1. 类型为 `int32[4][2]`，总字节数为 `32`。
2. 存储顺序为行优先连续存储，即先按频点递增，再在同一频点内写方位角和俯仰角。
3. 索引定义如下：
   `freqBeamCoord[0][0]` 为频点 1 方位角，`freqBeamCoord[0][1]` 为频点 1 俯仰角。
   `freqBeamCoord[1][0]` 为频点 2 方位角，`freqBeamCoord[1][1]` 为频点 2 俯仰角。
   `freqBeamCoord[2][0]` 为频点 3 方位角，`freqBeamCoord[2][1]` 为频点 3 俯仰角。
   `freqBeamCoord[3][0]` 为频点 4 方位角，`freqBeamCoord[3][1]` 为频点 4 俯仰角。
4. 本字段用于支持“四个频点可指向四个方向”的需求，允许多个频点写成同一组角度值，因此不采用位定义或索引表间接表达。
5. 当 `fieldValidFlags.Bit0 = 0` 时，接收端应忽略本数组，使用 `mainBeamCoord[2]` 作为默认指向。

### 4.3 频率代码 `freqCode`

1. 类型为 `uint8`，总字节数为 `1`。
2. 当前版本只使用 4 个固定频率代码，不再使用频率代码数组。
3. 代码对应关系如下：
   `0x01` 对应固定频率 1。
   `0x02` 对应固定频率 2。
   `0x03` 对应固定频率 3。
   `0x04` 对应固定频率 4。
4. 四频点波束阵面坐标中的 4 行与上述 4 个固定频率按顺序一一对应。
5. 当工作模式字中的频点组合不是四频点时，仅使用前若干个固定频率，其余频点对应的指向坐标写 `0`。

### 4.4 六增益控制数组 `rxGainCode[3][2]`

1. 类型为 `uint8[3][2]`，总字节数为 `6`。
2. 采用二维数组表示 `3` 个接收区和 `2` 个衰减器，不再拆成六个零散字段。
3. 存储顺序为行优先连续存储，先按接收区递增，再在同一接收区内写两个衰减器。
4. 索引定义如下：
   `rxGainCode[0][0]` 为接收区 1 的衰减器 1 增益代码。
   `rxGainCode[0][1]` 为接收区 1 的衰减器 2 增益代码。
   `rxGainCode[1][0]` 为接收区 2 的衰减器 1 增益代码。
   `rxGainCode[1][1]` 为接收区 2 的衰减器 2 增益代码。
   `rxGainCode[2][0]` 为接收区 3 的衰减器 1 增益代码。
   `rxGainCode[2][1]` 为接收区 3 的衰减器 2 增益代码。
5. 量化统一为 `0.5dB`，换算关系为：`实际增益 = 代码值 × 0.5dB`。
6. 若某一接收区当前未启用，对应两项应同时写 `0`。

### 4.5 噪声门限数组 `noiseThreshold[3]`

1. 类型为 `float[3]`，总字节数为 `12`。
2. 采用一维数组表示 `3` 个接收区的噪声门限。
3. 索引定义如下：
   `noiseThreshold[0]` 为接收区 1 噪声门限。
   `noiseThreshold[1]` 为接收区 2 噪声门限。
   `noiseThreshold[2]` 为接收区 3 噪声门限。
4. 当 `fieldValidFlags.Bit3 = 0` 时，执行侧不得把该数组解释为有效门限。
5. 若某一接收区当前未启用，对应位置建议写 `0.0`。

### 4.6 倾角补偿输入 `tiltCompInput[2]`

1. 类型为 `int32[2]`，总字节数为 `8`。
2. 索引定义如下：`tiltCompInput[0]` 为 X 轴倾角输入，`tiltCompInput[1]` 为 Y 轴倾角输入。
3. 量化统一为 `0.001deg`，有效范围沿用自然状态回传中的倾角口径。
4. 当 `tiltCompCtrl.Bit0 = 0` 时，执行侧不得使用该数组参与补偿计算。

### 4.7 校准后配相幅度数组 `phaseCalAmp[4][3]`

1. 类型为 `uint16[4][3]`，总字节数为 `24`。
2. 本数组按 `4` 个频点、`3` 个通道组织，用于保存以和通道为基准的校准后幅度结果。
3. 存储顺序为行优先连续存储，先按频点递增，再在同一频点内写三个通道。
4. 通道索引定义如下：
   `phaseCalAmp[f][0]` 为频点 `f+1` 的方位差通道幅度。
   `phaseCalAmp[f][1]` 为频点 `f+1` 的俯仰差通道幅度。
   `phaseCalAmp[f][2]` 为频点 `f+1` 的辅助通道幅度。
5. 量化采用 `0.1dB`，线上值 `15` 表示 `1.5dB`。
6. 当前实际使用前三个频点时，第 4 个频点整行保留，用于后续扩展；保留行未启用时写 `0`。

### 4.8 校准后配相相位数组 `phaseCalPhase[4][2]`

1. 类型为 `int16[4][2]`，总字节数为 `16`。
2. 本数组按 `4` 个频点、`2` 个差通道组织，用于保存以和通道为基准的相位结果。
3. 索引定义如下：
   `phaseCalPhase[f][0]` 为频点 `f+1` 的方位差通道相位。
   `phaseCalPhase[f][1]` 为频点 `f+1` 的俯仰差通道相位。
4. 辅助通道只输出幅度，不输出相位，因此不在本数组中占位，不增加无意义保留列。
5. 量化采用 `0.1deg`，线上值 `-35` 表示 `-3.5deg`。
6. 当 `fieldValidFlags.Bit1 = 0` 时，执行侧不得读取 `phaseCalAmp[4][3]` 和 `phaseCalPhase[4][2]`。

### 4.9 波位时间戳字段

1. 时间戳由 `timestampHour`、`timestampMinute`、`timestampSecond`、`timestampMillisecond` 四个字段组成，总字节数为 `5`。
2. 四个字段按“时、分、秒、毫秒”顺序连续存放。
3. 当 `fieldValidFlags.Bit7 = 0` 时，接收端不得把该时间戳解释为有效业务参数。

### 4.10 时间质量字段 `timeQuality`

1. 类型为 `uint8`，总字节数为 `1`。
2. 当 `fieldValidFlags.Bit6 = 0` 时，接收端不得把该字段解释为有效业务参数。
3. 枚举定义如下：
   `0` 表示 `Q0(Locked)`。
   `1` 表示 `Q1(Holdover)`。
   `2` 表示 `Q2(Freerun/Unknown)`。
4. 该字段仅描述当前时间质量等级，不替代独立的时间戳字段。

## 5. 工作、校准、检查定义

### 5.1 工作模式字 `workModeWord`

`workModeWord` 长度固定为 `1` byte，仅用于描述“工作态”下的执行位域，不再承载校准和检查语义。

| 位段 | 名称 | 定义 |
| -- | -- | -- |
| `Bit0` | 搜索跟踪选择 | `0` 表示搜索，`1` 表示跟踪 |
| `Bit2~Bit1` | 频点组合 | `00` 单频点，`01` 双频点，`10` 三频点，`11` 四频点 |
| `Bit4~Bit3` | 展宽模式 | `00` 不展宽，`01` 接收展宽，`10` 发射展宽，`11` 收发展宽 |
| `Bit5` | T/R 辐射使能 | `0` 关闭，`1` 开启 |
| `Bit6` | 匿影通道开关 | `0` 关闭，`1` 开启 |
| `Bit7` | 保留 | 当前为 `0` |

补充说明如下：

1. 当 `Bit2~Bit1` 取值为单频点、双频点或三频点时，仅使用前若干个固定频率，其余频点对应的指向坐标写 `0`。
2. 校准模式和检查模式不通过 `workModeWord` 复用表达，避免一个字节承载多种业务语义。

### 5.2 校准模式 `calibMode`

`calibMode` 采用单一模式枚举，定义如下：

| 取值 | 含义 |
| -- | -- |
| `0` | 无校准 |
| `1` | 外信号多频点校准 |
| `2` | 内信号四路校准源开启 |
| `3` | 内信号校准通道关闭 |
| `4~255` | 预留 |

补充说明如下：

1. 当 `calibMode = 1` 时，执行侧应关闭 T/R 辐射输出，并使用 `freqCode` 与 `calibLfmDelay` 共同描述外信号 LFM 校准参数。
2. 当 `calibMode = 2` 时，执行侧应打开四路校准源，配相数据区可作为该模式下的结果回写依据。
3. 当 `calibMode = 3` 时，表示当前波位用于关闭校准通道，不再产生新的校准输出。

### 5.3 检查模式 `inspectMode`

`inspectMode` 采用单一模式枚举，定义如下：

| 取值 | 含义 |
| -- | -- |
| `0` | 无检查 |
| `1` | 通信检查 |
| `2` | 内部电路检查 |
| `3` | 同步处理检查 |
| `4` | 方向图或 S 曲线检查 |
| `5` | 接收通道检查 |
| `6~255` | 预留 |

补充说明如下：

1. `通信检查` 默认包含误码率检查语义，同时要求覆盖通信链路涉及的全部数据区完整性检查。
2. `同步处理检查` 推荐用于开机后的时统同步确认。
3. `方向图或 S 曲线检查` 和 `接收通道检查` 均为单一模式，不与其他检查项组合置位。

## 6. 偏移计算规则与解析约束

### 6.1 倾角补偿控制字 `tiltCompCtrl`

| 位段 | 名称 | 定义 |
| -- | -- | -- |
| `Bit0` | 倾角补偿使能 | `0` 不启用，`1` 启用 |
| `Bit1` | 输入来源选择 | `0` 使用默认补偿，`1` 使用 `tiltCompInput[2]` |
| `Bit2` | 允许执行侧二次修正 | `0` 不允许，`1` 允许 |
| `Bit3` | 使用最新自然状态 | `0` 不强制，`1` 强制使用最新自然状态回传结果 |
| `Bit7~Bit4` | 保留 | 当前为 `0` |

### 6.2 字段有效标志字 `fieldValidFlags`

| 位段 | 名称 | 定义 |
| -- | -- | -- |
| `Bit0` | 四频点指向有效 | `0` 无效，`1` 有效 |
| `Bit1` | 配相数据有效 | `0` 无效，`1` 有效 |
| `Bit2` | 跟踪目标距离有效 | `0` 无效，`1` 有效 |
| `Bit3` | 噪声门限有效 | `0` 无效，`1` 有效 |
| `Bit4` | 倾角补偿输入有效 | `0` 无效，`1` 有效 |
| `Bit5` | 校准LFM时延有效 | `0` 无效，`1` 有效 |
| `Bit6` | 时间质量标记 | `0` 无效，`1` 有效 |
| `Bit7` | 时间戳有无标志 | `0` 无效，`1` 有效 |

### 6.3 偏移计算规则

1. 固定区长度为 `20` byte。
2. 第 `0` 个波位项固定从 `offset = 20` 开始。
3. 单个波位项固定长度为 `136` byte。
4. 第 `k` 个波位项内任一字段的实际偏移统一按下式计算：

`实际偏移 = 20 + 136×k + 字段相对偏移`

5. 整帧校验码偏移为 `20 + 136×N`。
6. 帧结束校验字偏移为 `24 + 136×N`。

### 6.4 解析约束

1. 接收端必须先完成整帧合法性校验，再进入 D 表业务字段解析。
2. 接收端必须优先读取 `波位中CPI计数`，据此计算波位项总长度和结束校验区位置。
3. 当 `fieldValidFlags` 对应位为 `0` 时，接收端不得把相关字段解释为有效业务参数。
4. 当前版本把未使用的固定区空间统一并入 `保留字段k`，不再赋予独立业务语义。
5. 当前版本进一步压缩了固定区和波位项控制区，只保留本轮评审确认需要的参数。
6. 当前版本保留波位项内分解式时间戳字段，按“时、分、秒、毫秒”顺序连续存放；当 `fieldValidFlags.Bit7 = 0` 时，接收端不得把该时间戳解释为有效业务参数。
7. 当前版本单个波位项补齐为 `136` byte，以保证各波位项起始地址保持 4 字节对齐。
