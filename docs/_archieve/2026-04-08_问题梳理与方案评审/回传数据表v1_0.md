# 回传数据表 v1.0

## 1. 适用范围

本文档用于定义前端设备向上位机回传的业务数据格式，包含以下两类独立包型：

1. `C1 自然状态回传`，用于周期性回传设备在线状态、工作状态、硬件健康状态、故障与告警信息，以及姿态与倾角传感器观测量，对应 `包类型 = 0x03`。
2. `C2 波位数据回传`，用于回传波位级采样结果，对应 `包类型 = 0x04`。

本文档采用与 [波束安排表v2_0.md](D:\WorkSpace\Company\Tower\rx_tech_demo\docs\2026-04-08_问题梳理与方案评审\波束安排表v2_0.md) 一致的表达方式，在同一份文档内分别使用两张完整总表描述 `C1` 和 `C2`。

## 2. C1 自然状态回传

### 2.1 整帧结构与长度关系

```text
C1 整帧结构
├─ 表头（固定 24 byte）
├─ C1状态包体（固定 52 byte）
└─ 包尾（固定 8 byte）
```

长度关系如下：

1. `表头长度 = 24`
2. `C1状态包体长度 = 52`
3. `包尾长度 = 8`

因此整包长度为：

`整包长度 = 24 + 52 + 8`

即：

`整包长度 = 84`

## 3. C2 波位数据回传

### 3.1 整帧结构与长度关系

```text
C2 整帧结构
├─ 表头（固定 24 byte）
├─ 数据控制区（固定 24 byte）
├─ IQ数据字节长度（固定 4 byte）
├─ IQ数据内容（变长）
└─ 包尾（固定 8 byte）
```

长度关系如下：

1. `表头长度 = 24`
2. `数据控制区长度 = 24`
3. `IQ数据字节长度字段 = 4`
4. `IQ数据内容长度 = iqDataLength`
5. `包尾长度 = 8`

因此整包长度为：

`整包长度 = 24 + 24 + 4 + iqDataLength + 8`

即：

`整包长度 = 60 + iqDataLength`

其中 `iqDataLength` 表示本包内 `IQ数据内容` 的有效字节长度。

## 4. 公共解析约束

1. 接收端必须先完成外层合法性校验，再依据 `包类型` 选择 `C1` 或 `C2` 解析路径。
2. 进入 `C1` 解析时，建议先检查：`包类型 = 0x03`、`状态区长度 = 52`、`整包长度 = 84`。
3. 进入 `C2` 解析时，建议先检查：`包类型 = 0x04`、`控制区长度 = 24`、`IQ数据字节长度` 与本包剩余长度是否一致。
4. `C1` 中的姿态观测字段建议按如下顺序解释：先检查 `倾角传感器状态字`，再检查 `倾角传感器有效标志字`，最后读取 `航向角`、`倾角传感器X`、`倾角传感器Y`。
5. `C2` 中的 `采样起始`、`采样长度`、`采样点数` 三者均采用 `sample` 口径，用于上位机对原始数据进行解释与还原。
6. `C2` 中 `IQ数据内容` 为变长字段，包尾偏移必须依据 `iqDataLength` 动态计算，不得按固定偏移解释。
