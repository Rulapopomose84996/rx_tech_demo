# 做一个Linux（amd64）环境下的模拟发包程序，

## 背景
为了方便做雷达数据流接收模块的测试，需要有一个固定节拍的UDP数据包发送程序。此发包程序运行在Ubuntu Linux（amd64）环境上。我会提供给你拟定好的数据包格式，你按照我规定好的节拍进行发送即可，实现轻量化一些，代码结构简单易懂，不要上复杂的设计模式和技术。

## 基本要求
1. 按照固定节拍发送拟好的UDP数据包（payload），见“data\cpi_0001_complete”。
2. 做发送和对于发送数据的检测，此外不开发其他功能。
3. 发送模式分为两种，一种是开启发射只有手动关闭才停；一种是设置固定发射时间，时间结束就自动停止发送。
4. 参数尽量支持config文件配置，方便后续调整。

## 基本信息
1. 发送周期为一个PRT = 119us, 假设第一个PRF起始为零时刻，我计划模拟0-19us进行发包（即19us是我们进行模拟的发送时间），之后暂停100us进入下一个PRT，下一个PRT依旧为0-19us进行发包，以此类推，直到结束。
2. 一个PRT内 单通道数据包为9个 共三个通道 共27个数据包 每个数据包大小为2048B 一个PRT内总数据量为27*2048B = 55296B = 54KB。具体参考：D:\WorkSpace\Company\Tower\rx_tech_demo\docs\设计方案\接收端专题\08_协议_波位控制与接收端补充说明.md
3. 在我们模拟的发送时间内（0-19us），每个数据包的发送时间点可以随机分布，但必须保证所有数据包在19us之前发送完成。算一下瞬时带宽为 BW = 54KB/19us = 2.91 GB/s；平均带宽为 BW = 54KB/119us = 453 MB/s，实际情况略大。
