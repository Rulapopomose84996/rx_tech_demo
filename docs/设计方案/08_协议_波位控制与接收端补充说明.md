## 3. 已明确的协议事实

### 3.1 采样与数据规模

#### 3.1.1 采样参数

* 三段信号回波采样门分别为：`4.8 us`、`16 us`、`70 us`
* 带宽：`25 MHz`
* 采样率：`50 MHz`

#### 3.1.2 单通道采样量

* 单通道采样点数：`240 + 800 + 3500 = 4540` 个 IQ
* 单通道数据量：`18160` 字节

#### 3.1.3 四通道总量

* 4 个通道总计：`18160` 个 IQ
* 总数据量：`72640` 字节

### 3.2 波位控制表

当前已明确：

* 长度：`2048` 字节
* 包头：`0x55AAFF00`
* CPI 计数：`2` 字节
* 其余字节：填 `0`

### 3.3 UDP 传输约定

#### 3.3.1 总体约定

* 单个 UDP 包长度：`2048` 字节
* 一个 PRT 需要发送：`9 × 4 = 36` 个 UDP 包
* 4 组 UDP 包依次对应：
* 和路数据
* 俯仰差数据
* 方位差数据
* 匿隐通道数据
* 每个通道占 `9` 个 UDP 包

#### 3.3.2 包头与包尾

* 包头：`0x55AAFF03`
* 包尾规则：
* 只有当前波位的最后一个 PRT 的最后一个 UDP 包带有效包尾 `0x55AAFF30`
* 其余 UDP 包包尾位置填充 `4` 字节 `0`

### 3.4 UDP 包结构

#### 3.4.1 包信息区

包信息固定 `16` 字节，字段如下：

| 偏移字节 | 长度 | 字段 | 说明 |
|---|---:|---|---|
| `0~3` | 4 字节 | 包头 | 固定 `0x55AAFF03` |
| `4~5` | 2 字节 | CPI 计数 | 范围 `0~65535` |
| `6~7` | 2 字节 | 通道计数 | `0/1/2/3` 分别对应和、俯仰差、方位差、匿隐通道 |
| `8~9` | 2 字节 | PRT 计数 | 当前 PRT 计数 |
| `10~11` | 2 字节 | 包计数 | 范围 `0~8` |
| `12~15` | 4 字节 | 包尾 | 最后一个有效包为 `0x55AAFF30`，其余填 `0` |

#### 3.4.2 数据区

* 数据区长度：`2048 - 16 = 2032` 字节
* 数据从偏移 `16` 开始
* 数据排列顺序：`I`、`Q`、`I`、`Q` ...
* 从图片表格可见：
* `16~17`：I
* `18~19`：Q
* `20~21`：I
* `22~23`：Q

### 3.5 每通道 9 包装载方式

单通道共 `4540` 个 IQ，对应 `9` 个 UDP 包：

* 前 `8` 个 UDP 包：每包 `508` 个 IQ
* 第 `9` 个 UDP 包：`476` 个 IQ
* 第 `9` 个包的剩余空间：全部填 `0`

校验关系如下：

* 单包数据区 `2032` 字节
* `508` 个 IQ 正好占满 `2032` 字节
* `8 × 508 + 476 = 4540`

### 3.6 联调时应统一的关键点

* 波位控制表长度固定为 `2048` 字节
* UDP 包长度固定为 `2048` 字节
* 一个 PRT 固定发送 `36` 个 UDP 包
* 通道顺序固定为：和、俯仰差、方位差、匿隐
* 通道计数与包计数字段长度均为 `2` 字节
* 包计数范围按当前记录为 `0~8`
* 仅“当前波位最后一个 PRT 的最后一个 UDP 包”带有效包尾
* 非有效数据区统一补 `0`

## 5. 接收端在这些事实之上的最小解释约束

在不超出上述协议事实的前提下，接收端主链路仍可保持以下最小解释约束：

* 只对已明确存在的字段做解析和校验
* 按 `(CPI, PRT, Channel, PacketIndex)` 形成 CPI 内部相对位置
* 基于当前已知的 `36` 包、`4` 通道、每通道 `9` 包组织固定槽位
* 对外最小输出单元仍为 CPI

若后续需要冻结字节序、计数递增规则或量化格式，应由冻结资产先行明确，再同步回写接收端专题文档。

本文不会改变接收端主链路的总体方向：
