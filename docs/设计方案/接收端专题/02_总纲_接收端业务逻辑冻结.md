# 接收端业务逻辑相关冻结

## 1. 主流程冻结

接收端主流程冻结为一条 owner 内顺序闭环：

```mermaid
flowchart TD
    A["接收会话准备"] --> B["网络入口收包"]
    B --> C["协议解析与单包合法性校验"]
    C --> D{"是否允许写入当前 CPI"}
    D -- "否" --> D1["owner 内计数并丢弃"]
    D -- "是" --> E["按 PRT / 通道 / 分片定位固定槽位"]
    E --> F{"目标槽位是否已写入"}
    F -- "是" --> F1["记为重复写入并结束"]
    F -- "否" --> G["一次落位到 CPI 固定槽位"]
    G --> H["更新位图 / 长度 / 状态 / 聚合摘要"]
    H --> I{"是否达到 CPI 收口条件"}
    I -- "否" --> B
    I -- "是" --> J["CPI 裁决、封口和输出准备"]
    J --> K["最终 CPI 输出对象"]
    K --> L["SPSC 交给下游"]

    B -.轻量统计 / 事件.-> M["异步旁路能力"]
    C -.轻量统计 / 事件.-> M
    D1 -.轻量统计 / 事件.-> M
    F1 -.轻量统计 / 事件.-> M
    J -.轻量统计 / 事件.-> M
    K -.必要副本 / 摘要.-> M
```

## 2. 归属判定冻结

归属阶段的主任务不是解释所有失败类型，而是回答两个问题：

* 这个包是否合法且可解释
* 这个包是否允许写入当前 CPI

因此归属阶段统一收敛为：

```mermaid
flowchart TD
    A["输入：已通过单包合法性校验的报文"] --> B["读取当前有效波位安排快照"]
    B --> C{"波位安排是否可用"}
    C -- "否" --> C1["写轻量 reason code 并丢弃"]
    C -- "是" --> D["提取 CPI / PRT / Channel / PacketIndex"]
    D --> E["按冻结计数规则判断是否命中当前 CPI"]
    E --> F{"是否允许写入当前 CPI"}
    F -- "否" --> F1["写轻量 reason code 并丢弃"]
    F -- "是" --> G["形成 CPI 内部相对位置"]
    G --> H["输出归属成功结果"]
```

细的“过期 / 未来 / 越界 / 不可归属”原因可以保留，但统一降级为内部 reason code。

## 3. 重组与聚合冻结

重组与聚合统一冻结为：

* CPI 是最终聚合边界
* PRT / 通道 / 分片只用于内部定位与进度推进
* 分片归属成功后一次落位到固定槽位
* 后续不再多次复制原始 IQ 数据

```mermaid
flowchart TD
    A["输入：归属成功结果"] --> B["在 owner 本地总表定位 CPI 上下文"]
    B --> C["按 PRT / Channel / PacketIndex 定位固定槽位"]
    C --> D{"是否首次写入"}
    D -- "否" --> D1["重复写入计数并结束"]
    D -- "是" --> E["payload 一次性落位"]
    E --> F["更新分片位图和长度"]
    F --> G["更新通道、PRT、CPI 聚合摘要"]
    G --> H{"是否达到 CPI 收口条件"}
    H -- "否" --> I["继续等待后续分片"]
    H -- "是" --> J["进入 CPI 裁决阶段"]
```

## 4. 观测与旁路冻结

主链路指标只保留高价值核心集合：

* 收包速率
* 字节速率
* 非法包计数
* 重复包计数
* 丢弃计数
* CPI 完整提交数
* CPI 不完整提交数
* CPI 作废数

统计、日志、录制、旁路的统一约束是：

* 异步
* 从属
* 不反压
* 不侵入主链路对象模型

---

## 5. 本文结论

接收端业务逻辑已经统一收敛为：

* 以 CPI 为输出边界
* 以 owner 为唯一主执行者
* 以固定槽位一次落位为数据组织方式
* 以最少必要判断保护当前 CPI
* 以轻量指标和从属旁路支撑联调与观测
