## 总体冻结结论

按“当前环境下，只追求主链路性能、确定性和可收敛性”的目标，接收端冻结为以下主形态：

> DPDK 原生 `i40e` PMD + `vfio-pci` 接管数据面网口 + 单 owner 线程 run-to-completion + 每分片一次落位到 CPI owner 固定槽位 + CPI 上下文单线程独占写入 + 最终 CPI 输出对象通过 SPSC 交给下游。

同时明确：

> 环境基线描述的是当前现场运行状态；架构冻结描述的是目标主链路形态；两者不得混写为同一层结论。

---

## 1. 主链路冻结口径

### 1.1 对外输出单元

接收端对外最小输出单元冻结为 CPI。

这意味着以下内容都必须按 CPI 收敛：

* 重组结果的最终归并边界
* 完整性判定
* 提交边界
* 封口和回收
* 对外统计口径

PRT / 通道 / 分片继续存在，但只作为 CPI 内部组织粒度。

### 1.2 线程与执行模型

冻结为单 owner 线程 run-to-completion：

* 收包
* 解析
* 单包合法性校验
* 归属判定
* 固定槽位落位
* 进度推进
* CPI 裁决、封口和输出准备

都在同一 owner 线程内顺序完成。

只有最终 CPI 输出对象允许跨线程交接。

### 1.3 数据组织方式

冻结为“固定槽位一次落位”：

* 归属成功后，分片 payload 一次性拷贝到 CPI 固定槽位
* 后续只维护位图、长度、状态和引用关系
* 不再维护通道级、PRT 级、CPI 级多份原始 IQ 大副本

### 1.4 主链路判断复杂度

主链路只做最少必要判断：

* 合法 / 非法
* 可写当前 CPI / 不可写当前 CPI
* 首次写入 / 重复写入

如需更细解释，只能降级为 reason code 或旁路调试口径，不再把细异常分类塑造成架构主角。

### 1.5 观测与旁路

主链路观测冻结为少量核心指标优先：

* 收包速率
* 字节速率
* 丢包或丢弃计数
* 非法包计数
* 重复包计数
* CPI 完整提交数
* CPI 不完整提交数
* CPI 作废数
* owner 线程与 NUMA 健康指标

日志、录制、详细追踪、旁路导出一律视为从属能力，不得反压主链路。

---

## 2. 技术栈冻结

### 2.1 收包技术栈

主数据面冻结为 `DPDK 原生 i40e PMD`，不采用 AF_XDP 作为主方案。

### 2.2 网口占有方式

`receiver0/1/2` 由 DPDK 独占并绑定到 `vfio-pci`。
管理口继续走 Linux 网络栈。

### 2.3 队列模型

主数据路径只保留最少必要队列。
默认只在“最终 CPI 输出对象 -> 下游消费者”这一处使用 SPSC。

### 2.4 NUMA 与绑定

数据面 owner、对象池和 CPI 上下文固定在 Node 1。
管理、旁路、录制优先放在非关键核或 Node 0。

---

## 3. 上下文与生命周期冻结

### 3.1 CPI ownership

同一 CPI 上下文从创建到封口只允许一个 owner 写。

### 3.2 生命周期最小状态

最小状态集合冻结为：

* `ACTIVE`
* `DECIDING`
* `SEALED`
* `TOMBSTONE`
* `RECYCLED`

### 3.3 迟到包处理

命中 `SEALED / TOMBSTONE` 的同键包稳定判为过期包，不得重建旧 CPI 上下文。

---

## 4. 裁决与输出冻结

### 4.1 裁决角色

CPI 的完成判定、截止判定、封口与回收，由同一个 owner 线程负责。

### 4.2 裁决结果

裁决结果统一冻结为四类：

* `COMPLETE_OK`
* `INCOMPLETE_BUT_COMMITTABLE`
* `ABNORMAL_CUTOFF_COMMIT`
* `DISCARD_INVALID`

### 4.3 输出路径

只有前三类允许生成最终 CPI 输出对象。
下游只读取最终对象，不回写接收上下文。

---

## 6. 本文结论

接收端主链路冻结后的关键词只有六个：

* `CPI`
* `owner`
* `fixed-slot`
* `one-copy`
* `minimal-judgment`
* `lightweight-observability`

目录内其他文档必须围绕这六个关键词对齐表述。
