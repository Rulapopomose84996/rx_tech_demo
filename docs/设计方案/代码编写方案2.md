### Phase B — 配置文件优先重构

**目标**：CLI 只保留 `--config / --dry-run / --help`，所有运行参数通过 conf 文件传递。配置文件格式升级为带 section 的 INI 风格，零依赖。

**步骤**

**B-1. 扩展配置文件格式解析器**（rx_config.cpp）

在 `load_config_file()` 中增加 section 识别逻辑：
- 遇到 `[section_name]` 行时记录 current_section
- key 存储为 `section.key`（如 `network.interface_name`）
- 无 section 的 key 沿用现有逻辑，向后兼容旧 conf 文件
- 解析优先级：`network.interface_name` > 旧裸 key `interface_name`

支持的 sections 及归属：

| Section | 字段 |
|---------|------|
| `[dpdk]` | `pci_addr`, `port_id`, `socket_mem_mb`, `mempool_size`, `mbuf_cache_size`, `rx_desc`, `tx_desc` |
| `[network]` | `interface_name`, `queue_id`, `receiver_ipv4`, `allowed_source_ipv4`, `allowed_dest_port` |
| `[capture]` | `enabled`, `output_dir`, `index_filename`, `data_filename` |
| `[protocol]` | `udp_packet_size`, `channels_per_prt`, `packets_per_channel`（Phase C 使用）|
| `[runtime]` | `duration_seconds`, `run_until_stopped`, `max_burst`, `cpu_cores`, `status_interval_seconds` |
| `[log]` | `level`, `output`, `file_path` |
| `[feedback]` | `enabled`, `host`, `port`, `bind_host`, `interval_seconds` |

**B-2. 扩展 `RxConfig` 结构体**（rx_config.h）

新增字段：
```
// [log]
std::string log_level = "info";          // debug/info/warn/error
std::string log_output = "stdout";       // stdout / file
std::string log_file_path;

// [capture]
bool capture_enabled = true;
std::string capture_output_dir = "results";
std::string capture_index_filename = "capture_index.csv";
std::string capture_data_filename = "capture.bin";

// [protocol] — 供 Phase C ProtocolSpec 使用，先占位
uint32_t protocol_udp_packet_size = 2048;
uint32_t protocol_channels_per_prt = 3;   // 当前保持3通道
uint32_t protocol_packets_per_channel = 9;
```

移除歧义字段 `output_dir`（由 `capture_output_dir` 接替），兼容期两者共存，`output_dir` 非空时作为 `capture_output_dir` fallback。

**B-3. 精简 `CliArgs` 和 `parse_cli_args`**（cli_args.h, cli_args.cpp）

`CliArgs` 只保留：
```cpp
struct CliArgs {
    std::string config_path;   // --config FILE（必须提供）
    bool dry_run = false;      // --dry-run
    bool help = false;         // --help / -h
    bool valid = true;
    std::string error_message;
};
```

移除所有 `--output / --iface / --queue / --duration / --max-burst / --cores` 等。`--config` 在非 `--dry-run` 情况下必须提供，否则打印使用说明并退出。

**B-4. 更新 `app_main_common.cpp`**（[src/receiver/app/common/app_main_common.cpp](src/receiver/app/common/app_main_common.cpp)）

流程简化为：
```
parse_cli_args()
  → if --help: print_usage(); exit
  → load_default_config()
  → load_config_file(cli.config_path)   ← 直接替换，不再做 merge
  → if --dry-run: print_config(); exit
  → validate_config()                   ← 新增：必填字段检查
  → ReceiveContext{config, backend, metrics}
```

**B-5. 更新示例配置文件**（dpdk_single_face.conf, dpdk_receiver0_replay.conf）

补全 section 分组，增加 `[log]` / `[capture]` / `[protocol]` 占位注释块，作为完整参考模板。

### Phase C — 协议层解耦（ProtocolSpec 参数化）

**目标**：所有协议 magic、包尺寸、通道数等常量从 Parser/Validator/Interpreter 中提取到 `ProtocolSpec`，并与 `RxConfig` 对接。

**步骤**

**C-1. 新建 `ProtocolSpec`**（新文件 `src/receiver/protocol/rxtech/protocol_spec.h`）

```cpp
struct ProtocolSpec {
    uint32_t magic_control    = 0x55AAFF00;
    uint32_t magic_data       = 0x55AAFF03;
    uint32_t magic_tail       = 0x55AAFF30;
    uint32_t udp_packet_size  = 2048;
    uint32_t packet_header_size  = 16;
    uint32_t packet_data_size    = 2032;   // udp_packet_size - packet_header_size
    uint32_t channels_per_prt    = 3;      // 暂保持3；后续对齐4时改此处
    uint32_t packets_per_channel = 9;
    uint32_t iq_per_full_packet  = 508;
    uint32_t iq_per_last_packet  = 476;
    uint32_t control_table_size  = 2048;
};

// 工厂函数，从 RxConfig 中读取 [protocol] section 字段填充
ProtocolSpec protocol_spec_from_config(const RxConfig& cfg);
```

同时在 `rx_config.cpp` 实现 `protocol_spec_from_config()`。

**C-2. `SamplePacketParser` → `PacketParser`**（sample_packet_parser.h）

- 类改名：`SamplePacketParser` → `PacketParser`
- 视图改名：`SamplePacketView` → `ParsedPacketView`（与设计文档 doc 对齐）
- 构造函数注入 `const ProtocolSpec& spec`（默认构造保留，使用默认 `ProtocolSpec{}`）
- 移除内部 `constexpr` 魔数，改为读 `spec_.*`
- 精简 `ParsedPacketView`：移除 `ip_fragment_offset / more_ip_fragments / header_offset / frame_length / error_reason`（传输层细节），只保留业务字段：
  ```cpp
  struct ParsedPacketView {
      bool valid = false;
      PacketKind kind = PacketKind::unknown;
      uint16_t cpi = 0;
      uint16_t channel = 0;
      uint16_t prt = 0;
      uint16_t packet_index = 0;
      uint32_t tail = 0;
      const uint8_t* payload_ptr = nullptr;
      uint32_t payload_len = 0;
      uint64_t rx_tsc = 0;        // 来自 PacketDesc（DPDK硬件时间戳或rdtsc）
      RejectReason reject_reason = RejectReason::none;
  };
  ```
- `SamplePacketKind` → `PacketKind`（枚举值保持 `unknown / control_table / data_packet`）

**C-3. `SamplePacketValidator` → `PacketValidator`**（[src/receiver/protocol/rxtech/sample_packet_validator.h](src/receiver/protocol/rxtech/sample_packet_validator.h)）

- 构造函数注入 `const ProtocolSpec& spec`
- 将 `packet_size == 2048 / payload_len == 2032 / channel <= 3 / packet_index in [1,9]` 等硬编码改为 `spec_.udp_packet_size / spec_.packet_data_size / spec_.channels_per_prt / spec_.packets_per_channel`
- 新增 `RejectReason` 枚举（替代现在的 `std::string error_reason`）：`none / invalid_len / invalid_header / invalid_channel / invalid_packet_index / invalid_tail / invalid_field_combo`
- 输出改为 `PacketValidity { bool ok; RejectReason reason; }`

**C-4. `ProtocolSequenceInterpreter` 参数化**（protocol_sequence_interpreter.h）

- 构造函数注入 `const ProtocolSpec& spec`
- `27 = 3×9` 改为 `spec_.channels_per_prt * spec_.packets_per_channel`
- `iq_count == 476 / == 508` 改为 `spec_.iq_per_last_packet / spec_.iq_per_full_packet`
- 输出类型改名 `ProtocolPacketView` → `InterpretedPacketView`（或保留名称，只清理字段）；移除 `channel_name`（改为 sidecar 层做映射）

**C-5. 更新 `OwnerLoop` 中的实例化点**（owner_loop.cpp）

原本各处直接 `SamplePacketParser parser;` 改为：
```cpp
auto spec = protocol_spec_from_config(context.config);
PacketParser parser{spec};
PacketValidator validator{spec};
ProtocolSequenceInterpreter interpreter{spec};
```

**C-6. 更新 `UdpPayloadAssembler`**（udp_payload_assembler.h）

将 `UdpPayloadFrame` 中的 `ts_ns / port_id / queue_id / face_id` 移出，改为：这些字段由 DPDK ingress 层在 `PacketDesc` 中传递，组装器不负责携带元数据。`UdpPayloadFrame` 只保留：
```cpp
struct UdpPayloadFrame {
    std::vector<uint8_t> udp_payload;
    uint32_t source_ipv4_be = 0;
    uint32_t dest_ipv4_be = 0;
    uint16_t source_port = 0;
    uint16_t dest_port = 0;
};
```

### Phase D — 核心热路径骨架

**目标**：按设计文档（接收端模块接口与 ownership 边界、实现级对象模型）落地 CpiContext 内存模型及 SlotWriter/ProgressTracker/Admission/Finalizer 骨架，重构 OwnerLoop 为模块调用链。不含 SPSC/下游消费者。

**步骤**

**D-1. 新建 `src/receiver/storage/`**

- `rxtech/cpi_context.h`：
  - `CpiState` 枚举：`ACTIVE / DECIDING / SEALED / TOMBSTONE / RECYCLED`
  - `CpiDecision` 枚举：`COMPLETE_OK / INCOMPLETE_BUT_COMMITTABLE / ABNORMAL_CUTOFF_COMMIT / DISCARD_INVALID`
  - 编译期常量（在同一头文件中）：`N_PRT_MAX`（初版建议 64）、`N_CHANNELS = 3`（暂保持）、`N_PACKETS_PER_CHANNEL = 9`、`SLOT_STRIDE = 2048`、`SLOTS_PER_PRT = N_CHANNELS * N_PACKETS_PER_CHANNEL`
  - `PrtSummary`结构：`ch_pkt_bitmap[N_CHANNELS]`（uint16，9-bit）、`ch_recv_count[N_CHANNELS]`、`ready_channel_count`、`recv_packet_count`、`flags`（seen_tail | completed）
  - `CpiHotHeader`：`cpi_id, wave_id, expected_n_prt, expected_slot_count, received_slot_count, duplicate_count, first_rx_tsc, last_rx_tsc, seal_tsc, state, decision, trigger_bits`
  - `CpiContext`：前段 `CpiHotHeader + PrtSummary[N_PRT_MAX] + slot_valid_bytes[N_PRT_MAX * SLOTS_PER_PRT]`；后段 `payload[N_PRT_MAX * SLOTS_PER_PRT][SLOT_STRIDE]`
  - 槽位计算内联函数：`slot_index(prt, channel, packet_index)` → `(prt * N_CHANNELS + channel) * N_PACKETS_PER_CHANNEL + packet_index`

- `rxtech/cpi_context_pool.h`：
  - `CpiContextPool`：启动时 `new CpiContext[POOL_DEPTH]`；`acquire()` / `release(pool_index)` 简单位图分配，owner 线程独占，无锁
  - `POOL_DEPTH` 默认 4（实际场景最多同时1个ACTIVE + 少量DECIDING）

- `src/slot_writer.cpp` + `rxtech/slot_writer.h`：
  - `SlotWriteResult { bool first_write; bool duplicate; uint32_t slot_index; RejectReason reason; }`
  - `SlotWriter::write(CpiContext& ctx, const ParsedPacketView& pkt)`：
    1. 计算 `slot_index`
    2. bitmap 检查 `ch_pkt_bitmap[channel] & (1u << packet_index)` → duplicate
    3. 首次写入：`memcpy(ctx.payload[slot_index], pkt.payload_ptr, pkt.payload_len)`
    4. `slot_valid_bytes[slot_index] = pkt.payload_len`
    5. `bitmap |= (1u << pkt.packet_index)`
    6. 更新 `ch_recv_count / recv_packet_count / received_slot_count / first_rx_tsc / last_rx_tsc`

- `src/progress_tracker.cpp` + `rxtech/progress_tracker.h`：
  - `ProgressTracker::advance(CpiContext& ctx, uint16_t prt, uint16_t channel)`
  - 检查 `PrtSummary[prt].ch_recv_count[channel] == N_PACKETS_PER_CHANNEL` → 更新 `ready_channel_count`
  - 检查 `ready_channel_count == N_CHANNELS` → PRT complete，更新 `ready_prt_count`
  - 检查 `received_slot_count == expected_slot_count` → 设置 `trigger_bits |= FULL_READY`
  - 检查 tail 包到达 → 设置 `trigger_bits |= TAIL_OBSERVED`

**D-2. 新建 `src/receiver/admit/`**

- `rxtech/cpi_admission.h`：
  - `AdmissionResult` 枚举：`WRITE_ACTIVE / TRIGGER_CPI_SWITCH / LATE_TO_CLOSED / DROP`
  - `RecentClosedEntry { uint16_t cpi_id; uint64_t seal_tsc; CpiDecision decision; }`
  - `RecentClosedRing`：固定深度 8，循环写入，`contains(cpi_id)` 查询
  - `CpiAdmission::judge(const ParsedPacketView& pkt, uint16_t active_cpi, const RecentClosedRing& closed)` → `AdmissionResult`：
    - `pkt.cpi == active_cpi` → `WRITE_ACTIVE`
    - `pkt.cpi == (active_cpi + 1) % 65536` → `TRIGGER_CPI_SWITCH`
    - `closed.contains(pkt.cpi)` → `LATE_TO_CLOSED`
    - else → `DROP`

**D-3. 新建 `src/receiver/output/`**

- `rxtech/cpi_output.h`：
  ```cpp
  struct CpiReadOnlyView {
      const uint8_t* payload_base;
      const uint16_t* slot_valid_bytes;
      const PrtSummary* prt_summary;
      uint32_t n_prt;
      uint32_t slot_count;
      uint32_t payload_stride;  // 2048
  };
  struct CpiOutput {
      uint16_t cpi_id;
      CpiDecision decision;
      uint32_t received_slot_count;
      uint32_t missing_slot_count;
      uint32_t duplicate_count;
      uint32_t ready_prt_count;
      uint64_t first_rx_tsc, last_rx_tsc, seal_tsc;
      CpiReadOnlyView view;
      uint32_t pool_index;   // 归还 ctx_pool 使用
      uint64_t output_id;
  };
  struct ReleaseToken {
      uint64_t output_id;
      uint32_t ctx_pool_index;
  };
  ```
- `rxtech/output_pool.h`：固定深度对象池，`acquire()` / `release(ReleaseToken)` 无锁位图分配

**D-4. `CpiFinalizer` 骨架补充**（finalize，目录已存在）

- `rxtech/cpi_finalizer.h`：
  - `CpiFinalizer::try_finalize(CpiContext& ctx, uint32_t trigger)` → `std::optional<CpiOutput>`
  - 触发条件判断：`FULL_READY` → `COMPLETE_OK`；`CPI_SWITCH` 或 `WAVE_END` → 视 ready_prt_count 决定 `INCOMPLETE_BUT_COMMITTABLE` 或 `ABNORMAL_CUTOFF_COMMIT`
  - `TIMEOUT / STOP` → 桩实现（返回 `nullopt`，留 TODO 注释）
  - 状态转换：`ACTIVE → DECIDING → SEALED`；`seal_tsc = rdtsc()`
  - 填充 `CpiOutput`，更新 `RecentClosedRing`
  - ctx 状态设为 `TOMBSTONE`（不直接回收，等待下游归还 ReleaseToken）

**D-5. 重构 `OwnerLoop`**（owner_loop.cpp）

将现有平铺逻辑拆分为调用链。`OwnerState` 新结构（内部于 `owner_loop.cpp`）：

```
OwnerState {
    uint16_t active_cpi;
    CpiContext* active_ctx;          // 来自 ctx_pool
    CpiContextPool ctx_pool;
    RecentClosedRing closed_ring;
    OutputPool output_pool;
    // 统计计数器（精简版）
    uint64_t total_rx, valid_packets, duplicates, drops, complete_cpis;
}
```

热循环核心逻辑改为：
```
for each pkt in burst:
    ParsedPacketView view = parser.parse(pkt);
    if (!view.valid) { metrics.on_invalid(); continue; }

    PacketValidity validity = validator.validate(view);
    if (!validity.ok) { metrics.on_reject(validity.reason); continue; }

    AdmissionResult admit = admission.judge(view, state.active_cpi, state.closed_ring);
    if (admit == TRIGGER_CPI_SWITCH):
        auto output = finalizer.try_finalize(*state.active_ctx, CPI_SWITCH);
        if (output) dispatch(*output, state);   // 放入 output_pool，更新closed_ring
        state.open_new_cpi(view.cpi, ctx_pool);
    else if (admit != WRITE_ACTIVE):
        metrics.on_drop(admit); continue;

    SlotWriteResult wr = writer.write(*state.active_ctx, view);
    if (!wr.first_write) { metrics.on_duplicate(); continue; }

    tracker.advance(*state.active_ctx, view.prt, view.channel);

    if (state.active_ctx->trigger_bits & FULL_READY):
        auto output = finalizer.try_finalize(*state.active_ctx, FULL_READY);
        if (output) dispatch(*output, state);
        state.open_new_cpi_or_idle();
```

**D-6. 精简 `IMetricsCollector` 和 `AsyncSidecar`**（[src/receiver/sidecar/rxtech/metrics.h](src/receiver/sidecar/rxtech/metrics.h)）

移除：`on_port_packet / on_reassembled_block / on_missing_fragments / on_duplicate_fragment / on_invalid_header`

保留/新增：
```cpp
virtual void on_burst(uint32_t count, uint64_t bytes) = 0;
virtual void on_valid_packet(PacketKind kind) = 0;
virtual void on_reject(RejectReason reason) = 0;
virtual void on_duplicate() = 0;
virtual void on_drop(AdmissionResult reason) = 0;
virtual void on_cpi_output(const CpiOutput& output) = 0;  // CPI 完成事件
virtual RunSummary finalize() = 0;
```

`RunSummary` 协议相关部分精简：`ProtocolChannelSummary` 只留 `{channel, data_packets, iq_count}`；`ProtocolCpiSummary` 只留 `{cpi_id, data_packets, prt_count, decision}`。
