# D表 波束安排表数据表文档

## 1. 适用范围

D表用于定义前端执行侧的波位级安排信息，用于描述一次安排批次内各波位的执行顺序、工作方式、指向参数、频点代码索引、三脉冲增益参数、采样参数、波位拼接参数以及波位内积累参数。

D表作为业务包体使用，外层统一采用 B 表通用格式封装。

---

## 2. 设计原则

### 2.1 总体原则

1. D表采用 **表头 + 波位项** 的组织方式。
2. 表头只承载安排级元信息，不承载传输层控制字段。
3. 波位项采用固定长度，便于 FPGA 侧 RAM 预分配、状态机解析和 DMA 访问。
4. 全表采用小端编码。
5. 全表按 4B 自然对齐设计。

### 2.2 去冗余原则

1. 不重复定义 B 表已有的包序号、包类型、包长、分包字段。
2. 不重复复制 A 表中一次性装订的完整波形定义。
3. 频点代码采用 **索引引用** 的方式，减少波位项字节占用。
4. 不使用“执行快照”式冗余字段。

### 2.3 本版边界说明

1. D表中保留时间戳字段，但本版时间戳仅定义为 **时/分/秒/ms** 的分解式时间戳。
2. 本版不在 D 表中引入“逻辑周期号”这类强软件调度语义字段。
3. D 表中的“波位内积累个数”表示当前波位计划执行的积累数量，不等同于 C 表中的“波位内积累序号”。
4. D 表不再定义“生效方式”字段，统一采用“探测周期边界下一周期生效”规则。

---

## 3. 总体结构

D表采用如下结构：

```text
D表 波束安排表
├─ 表头（固定 16B）
└─ 波位项（固定 32B，按本表波位数重复）
```

长度关系如下：

**D表长度 = 16B + N × 32B**

其中 `N` 为本表波位数。

---

## 4. 表头定义

## 4.1 表头结构

表头固定长度为 **16B**。

## 4.2 表头字段表

| 偏移 | 字段名称 | 数据类型 | 长度 | 说明 |
| -- | -- | -- | -: | -- |
| 0  | 表标识 | `uint16` | 2B | 标识当前为 D 表，固定值由系统统一分配 |
| 2  | 表头长度 | `uint16` | 2B | 固定值 `16` |
| 4  | 波位项长度 | `uint16` | 2B | 固定值 `32` |
| 6  | 本表波位数 | `uint16` | 2B | 当前表内波位项个数 |
| 8  | 波形组编号 | `uint16` | 2B | 引用 A 表中已装订的波形组编号 |
| 10 | 表结构版本 | `uint16` | 2B | 当前冻结版本为 `0x0101` |
| 12 | 预留字段 | `uint32` | 4B | 当前版本固定填 `0` |

## 4.3 表头字段说明

### 4.3.1 本表波位数

用于指示当前 D 表中实际包含的波位项数量。接收端应依据该字段解析后续波位项个数。

### 4.3.2 波形组编号

用于引用 A 表中已一次性装订的波形组资源。这样 D 表只负责“安排与调用”，不再复制一整套慢变波形定义。

### 4.3.3 表结构版本

当前版本冻结如下：

| 取值 | 含义 |
| -- | -- |
| `0x0100` | 历史草案版本，含“生效方式”字段 |
| `0x0101` | 当前冻结版本，删除“生效方式”字段，统一边界生效规则 |

### 4.3.4 生效规则

本版统一冻结如下生效规则：

1. 新下发 D 表在**探测周期边界**切换生效。
2. 生效动作定义为：校验通过后先写入 Shadow Buffer，待下一探测周期边界到达时切换为 Active Buffer。
3. 切换后从**新表第 0 个波位项**开始执行。
4. 旧表若正在执行中的波位内积累过程被边界切换截断，按“旧表本周期结束、新表下周期起始”的方式处理，不保留跨表延续语义。

### 4.3.5 生效时序示意

```mermaid
sequenceDiagram
    participant Host as 上位机
    participant Link as B表封装链路
    participant Exec as 执行侧

    Host->>Link: 下发 D 表
    Link->>Exec: 判帧 / CRC / 组包完成
    Exec->>Exec: 写入 Shadow Buffer
    Note right of Exec: 等待探测周期边界
    Exec->>Exec: 到达下一探测周期边界
    Exec->>Exec: Active/Shadow 切换
    Exec->>Exec: 从新表第 0 波位项开始执行
```

## 5. 波位项定义

## 5.1 波位项结构

每个波位项固定长度为 **32B**。

## 5.2 波位项字段表

> **设计说明**：采用数组化结构设计脉冲相关参数，将三脉冲的采样点数和 MGC 增益分别组织为数组，通过索引区分不同脉冲；索引体系统一冻结为 `PulseIndex = 0..2`。

| 偏移 | 字段名称 | 数据类型 | 长度 | 说明 |
| -- | -- | -- | -: | -- |
| 0 | 波位号 | `uint16` | 2B | 当前波位编号，在同一张 D 表内保持唯一 |
| 2 | 工作方式字 | `uint8` | 1B | 工作/检查、多单频、搜跟、展宽、T/R 辐射控制（位定义见 7.2 节） |
| 3 | 周期组合个数 | `uint8` | 1B | 周期组合个数，表示该波位的周期重复次数 |
| 4 | 方位角 | `int16` | 2B | 当前波位方位角，单位 `0.0025°` |
| 6 | 俯仰角 | `int16` | 2B | 当前波位俯仰角，单位 `0.0025°` |
| 8 | 脉冲采样点数[3] | `uint16[3]` | 6B | `PulseIndex=0..2` 对应三脉冲采样点数 |
| 14 | 采样起始 | `uint16` | 2B | 采样起始位置，单位为采样点 |
| 16 | 采样长度 | `uint16` | 2B | 采样持续长度，单位为采样点 |
| 18 | 频点代码索引 | `uint8` | 1B | 引用 A 表中已装订的三脉冲频点组合索引 |
| 19 | 脉冲 MGC 增益[3] | `uint8[3]` | 3B | 单位 `0.5 dB/LSB`，`PulseIndex=0..2` |
| 22 | 波位拼接个数 | `uint8` | 1B | 当前波位的拼接个数 |
| 23 | 波位内积累个数 | `uint8` | 1B | 当前波位计划执行的积累数量，范围 `1..255` |
| 24 | 时间戳-时 | `uint8` | 1B | 时间戳中的“时”（0~23） |
| 25 | 时间戳-分 | `uint8` | 1B | 时间戳中的“分”（0~59） |
| 26 | 时间戳-秒 | `uint8` | 1B | 时间戳中的“秒”（0~59） |
| 27 | 时间戳-毫秒高字节 | `uint8` | 1B | 时间戳中毫秒值的高 8 位 |
| 28 | 时间戳-毫秒低字节 | `uint8` | 1B | 时间戳中毫秒值的低 8 位（0~999） |
| 29 | PulseIndexBase | `uint8` | 1B | 固定填 `0`，明确数组索引基准为 0-based |
| 30 | PulseCount | `uint8` | 1B | 固定填 `3`，表示当前版本采用三脉冲 |
| 31 | Flags | `uint8` | 1B | 扩展标志字，当前版本固定填 `0x00` |

**对齐说明**：

- 偏移 0-7：波位号(2B) + 工作方式字(1B) + 周期组合个数(1B) + 方位角(2B) + 俯仰角(2B) = 8B
- 偏移 8-13：脉冲采样点数[3](6B)，起点为偶地址
- 偏移 14-17：采样起始(2B) + 采样长度(2B) = 4B
- 偏移 18-21：频点代码索引(1B) + 脉冲 MGC 增益[3](3B) = 4B
- 偏移 22-23：波位拼接个数(1B) + 波位内积累个数(1B) = 2B
- 偏移 24-28：时间戳(5B) = 时(1B) + 分(1B) + 秒(1B) + 毫秒(2B)
- 偏移 29-31：索引基准/脉冲个数/扩展标志(3B)

**总长度**：32B（严格固定，便于 FPGA RAM 预分配）

## 6. 波位项字段说明

### 6.1 波位号

用于标识当前安排项所属波位。
建议在同一张 D 表内保持唯一。

### 6.2 周期组合个数

该字段对应当前要求中的周期组合个数。
允许值冻结为：

- `2`
- `4`
- `6`
- `8`
- `12`
- `16`

其余取值当前版本视为非法值。

### 6.3 方位角、俯仰角

当前版本统一冻结如下：

1. 数据类型：`int16`
2. 量化单位：`0.0025°`
3. 实际角度计算公式：**实际角度 = 编码值 × 0.0025°**

字段表与说明段落均以此为准，不再保留 `0.01°` 口径。

### 6.4 脉冲采样点数[3]

用于分别描述三脉冲对应的采样点数。

索引规则统一为：

1. `PulseIndex = 0` 对应第一脉冲
2. `PulseIndex = 1` 对应第二脉冲
3. `PulseIndex = 2` 对应第三脉冲

### 6.5 采样起始、采样长度

统一采用“采样点”口径，而不采用 `μs` 口径。

这样更贴近硬件表、寄存器表和回传表的实现方式，也更方便与 C 表的数据控制表做一致性回显。

### 6.6 频点代码索引

本字段定义为**三脉冲频点组合索引**，而不是单一频点索引。

解释方式冻结如下：

1. `频点代码索引 = 0`：引用第 0 组频点代码数组
2. `频点代码索引 = 1`：引用第 1 组频点代码数组
3. 以此类推

每一组频点代码数组都对应三脉冲频点配置。

### 6.7 脉冲 MGC 增益[3]

当前版本统一冻结为：

1. 数据类型：`uint8[3]`
2. 量化单位：`0.5 dB/LSB`
3. 实际增益计算公式：**实际增益 = 编码值 × 0.5 dB**
4. 索引规则统一为 `PulseIndex = 0..2`

这样与 A 表中的手动增益口径保持一致。

### 6.8 波位拼接个数

用于描述当前波位安排中采用的拼接个数。

### 6.9 波位内积累个数

该字段用于描述当前波位计划执行的积累数量。
它是**安排参数**，与 C 表中的“波位内积累序号”属于不同语义。

### 6.10 时间戳（时/分/秒/ms）

本版将时间戳拆为四个字段：

- 时：`uint8`
- 分：`uint8`
- 秒：`uint8`
- 毫秒：`uint16`

当前版本仅冻结“字段存在”和“字段组成”，不追加更强的调度语义。

### 6.11 PulseIndexBase、PulseCount、Flags

这三个字节用于替代历史版本中的纯预留填充：

1. `PulseIndexBase` 固定填 `0`
2. `PulseCount` 固定填 `3`
3. `Flags` 当前固定填 `0x00`

这样在保持总长度不变的前提下，明确给出多脉冲索引元数据。

---

## 7. 工作方式字位定义

## 7.1 字段长度

`工作方式字` 长度固定为 **1B**。

## 7.2 位定义

| 位段 | 含义 | 定义 |
| -- | -- | -- |
| `D1-D0` | 工作类别 | `00` 工作，`01` 校准，`10` 测试，`11` 预留 |
| `D2` | 频点方式 | `0` 单频，`1` 多频 |
| `D4-D3` | 搜跟方式 | `00` 搜索，`01` 跟踪，`10` 搜跟混合，`11` 预留 |
| `D6-D5` | 展宽方式 | `00` 不展宽，`01` 接收展宽，`10` 发射展宽，`11` 收发展宽 |
| `D7` | T/R 辐射使能 | `0` 关闭，`1` 开启 |

---

## 8. 频点代码索引定义

## 8.1 字段作用

`频点代码索引` 用于引用系统内预置的三脉冲频点代码数组，而不在波位项中分别写入三组频点代码。

## 8.2 建议解释方式

建议系统维护一张 **频点代码数组表**，每一项包含：

1. 第 1 脉冲频点代码
2. 第 2 脉冲频点代码
3. 第 3 脉冲频点代码

D 表只下发索引号，前端执行时根据索引号查表取得对应三脉冲频点代码。

---

## 9. 长度关系

## 9.1 固定长度项

| 项目 | 长度 |
| -- | --: |
| 表头长度 | 16B |
| 波位项长度 | 32B |

## 9.2 总长度公式

**D表总长度 = 16B + 本表波位数 × 32B**

## 9.3 最小长度

当本表波位数为 `1` 时：

**D表最小长度 = 16B + 1 × 32B = 48B**

---

## 10. 建议冻结结论

本版 D 表冻结如下：

1. D表作为波束安排表，采用“表头 + 固定波位项”结构。
2. 表头固定长度为 `16B`。
3. 波位项固定长度为 `32B`。
4. D表不重复定义 B 表已有的传输层字段。
5. `工作方式字` 采用 `1B` 位定义方式。
6. `频点代码` 采用三脉冲组合索引方式。
7. 三脉冲 `MGC` 增益采用数组字段，量化为 `0.5 dB/LSB`。
8. 三脉冲采样点数采用数组字段。
9. “CPI 计数”在 D 表中正式改写为“波位内积累个数”。
10. 时间戳采用“时/分/秒/ms”分解式字段。
11. 角度字段统一采用 `int16 + 0.0025°` 量化方式。
12. 采样起始与采样长度统一采用采样点口径。
13. 表头不再包含“生效方式”字段，改为“表结构版本”字段。
14. D 表统一采用“探测周期边界下一周期生效”规则。
15. `PulseIndex` 统一为 `0..2`，采用 0-based 编号。
