# A表 一次性装订表文档

## 1. 适用范围

A 表用于服务器对前端设备进行**一次性下装**的慢变资源与模板参数，下装内容包括：

1. 三脉冲波形模板参数（量化描述）及其对应的存储资源映射。
2. 接收与采样链路的模板配置（采样门、滤波器、触发等）。
3. 资源区引用、滤波器版本与故障保护策略。

---

## 2. 设计原则

### 2.1 表间职责边界

A、D、C 三表之间的覆盖关系冻结如下：

| 表 | 职责 | 覆盖方向 |
| -- | -- | -- |
| **A 表** | 一次性装订，提供慢变默认模板与资源映射 | 装订后作为全局默认值生效 |
| **D 表** | 波位级安排，按波位覆盖 A 表模板中的执行参数 | D 表覆盖 A 表默认值 |
| **C 表** | 回传执行结果（回波数据、状态数据） | 只读，不反写 A 表 |

- **A 表给默认模板**：频点代码、MGC 增益、采样点数、存储映射等一次性定义；
- **D 表按波位覆盖**：可在每个波位项中替换频点、增益、采样参数；
- **C 表按实际结果回传**：回传的数据与 A 表无逻辑绑定，也不会修改 A 表内容。

### 2.2 波形生成模型

A 表只携带**波形模板参数**（量化值）与**波形资源映射**（存储地址），不直接携带精细波形数据。波形执行流程如下：

1. **服务器下装**：将脉冲体制代码、频点、带宽、脉宽等模板量化参数，以及脉冲波形在前端存储区内的地址和长度，通过 A 表一次性写入前端。
2. **前端执行**：依据存储映射中的地址定位实际波形资源（由离线串口或远程方式提前烧录），按 D 表的波位安排调用对应脉冲。
3. **量化字段用途**：A 表内的频点、带宽、脉宽等编码字段用于模板核对、离线分析和接口一致性校验，不直接替代存储资源作为执行依据。

### 2.3 极简原则

A 表只保留**慢变且资源级**的字段。执行语义类字段（工作类别、搜跟方式、波束展宽、发收使能等）均应在 D 表中按波位配置，不在 A 表中定义默认态。

### 2.4 编码原则

1. 全表采用**小端编码**。
2. 全表按 **4B 自然对齐**设计。
3. 多脉冲字段统一采用**数组化结构**，索引冻结为 `PulseIndex = 0..2`（0-based，0 为第一脉冲）。
4. 物理量量化口径与[前端感知通信协议 V1.0](docs/2026-04-07_问题梳理与方案评审/前端感知通信协议V1.0.md) 保持一致。

## 3. 完整包结构

A 表业务内容使用 B 表通用格式封装，完整包结构如下：

```text
┌──────────────────────────────────────────┐
│ B 表通用包头（固定 24B）                  │
│   帧首校验字 55AAAA55 / 包类型 / 包序号  │
├──────────────────────────────────────────┤
│ A 表业务体（固定 88B）                    │
│  ├─ 业务头（8B）                         │
│  ├─ 三脉冲波形模板区（24B）              │
│  ├─ 接收与采样模板区（16B）              │
│  ├─ 资源与保护配置区（8B）               │
│  └─ 存储映射区（32B）                    │
├──────────────────────────────────────────┤
│ B 表通用包尾（固定 8B）                   │
│   整帧 CRC32/IEEE / 帧尾校验字 AA5555AA  │
└──────────────────────────────────────────┘
```

**整包长度 = 24B + 88B + 8B = 120B**

B 表通用包头、包尾的字段定义见 B.md。A 表在 B 表中对应的包类型固定值由系统统一分配。

### 5.1 结构说明

业务头固定长度为 **8B**，仅承载 A 表业务层必要的最小标识信息。帧同步、包序号、分包等传输层控制字段均由 B 表通用包头承担，A 表业务头不重复定义。

## 6. 三脉冲波形模板区

### 6.1 结构说明

三脉冲波形模板区固定长度为 **24B**，按属性分组、按脉冲索引区分，采用数组化结构。

**本区字段用于量化描述波形模板属性，不直接执行波形输出。** 精细波形由前端固件依据存储映射区所指向的资源文件装载执行（见第 9 节）。各量化参数与[前端感知通信协议 V1.0](docs/2026-04-07_问题梳理与方案评审/前端感知通信协议V1.0.md) 物理量量化表保持一致，用于模板核对与接口对齐，不得与存储资源相互替代。

### 6.2 字段表

| 偏移 | 字段名称 | 数据类型 | 字长 | 取值范围 | 量化口径 | 注释说明 |
| -- | -- | -- | --: | -- | -- | -- |
| 8 | 脉冲体制代码[3] | `uint8[3]` | 3B | 每项 `0..255` | 无量纲编码 | **标识三路脉冲各自采用的波形体制或模板类别。**<br>该字段是存储资源映射的类型辅助标识，具体码值由系统统一枚举。例如可区分短脉冲（1μs级）、中脉冲（5μs级）、长脉冲（21μs级）等体制，与 时序说明.md 中 DSSP1/2/3 脉宽冻结值相对应。 |

### 6.3 补充说明

1. 本区字段总计 `3×6+6 = 24B`，不设额外预留字节，后续若新增脉冲属性，通过版本变更重组本区或追加扩展区。
2. 脉冲手动增益代码 [3] 与 D 表 `脉冲 MGC 增益[3]` 的覆盖关系：**A 表提供全局默认值；D 表按波位覆盖，优先级高于 A 表。**
3. 脉冲采样点数 [3] 与 D 表 `脉冲采样点数[3]` 的覆盖关系同上。

## 9. 存储映射区

### 9.1 结构说明

存储映射区固定长度为 **32B**，用于给出波形资源与滤波系数资源在前端存储空间内的落点。三脉冲相关字段统一采用数组化结构，通过 `PulseIndex` 区分不同脉冲的资源位置。

**本区是前端执行波形的最终依据**：前端固件依据 `脉冲体制代码[3]` 选择体制类型，依据本区的基地址和长度定位精细波形资源并装载执行，不依赖三脉冲模板区中的量化编码字段进行直接输出。

### 9.3 补充说明

1. 存储映射区所有字段按类型分组排列，确保 `uint32` 字段均处于 4B 对齐偏移（56、68、84），不插入额外对齐填充字节。
2. `脉冲波形存储基地址[3]` 与 `脉冲波形字节长度[3]` 描述的是资源文件或波形块的存储布局，不等同于波门时序参数。

## 12. 冻结结论

1. A 表业务体固定 **88B**，整包（含 B 表包头包尾）固定 **120B**。
2. 多脉冲字段统一采用数组化结构，索引冻结为 `PulseIndex = 0..2`（0-based）。
3. A 表只承载慢变资源与模板参数，执行语义字段（工作类别、搜跟、展宽、使能等）均由 D 表按波位定义。
4. 覆盖关系冻结：**A 表给默认模板 → D 表按波位覆盖 → C 表回传结果（不反写 A 表）**。
5. 波形执行依据：**存储映射区指向的硬件资源**为最终执行依据，三脉冲模板区量化字段用于模板核对与接口对齐。
6. 量化口径与 V1.0 通信协议保持一致，不引入新的量化单位变体。
